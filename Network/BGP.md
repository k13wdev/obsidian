## Суть и контекст

BGP версии 4 — протокол граничных маршрутизаторов, стандарт де-факто для маршрутизации между автономными системами в интернете. Часто называется протоколом, «склеивающим» глобальную сеть воедино — именно BGP позволяет тысячам независимых сетей взаимодействовать друг с другом.

---

## Ключевые идеи

- BGP работает между автономными системами (AS), а не внутри них
- Каждая AS идентифицируется глобально уникальным номером (ASN)
- BGP использует TCP (порт 179) — не требует собственных механизмов надёжности
- eBGP — между разными AS; iBGP — внутри одной AS
- Главные атрибуты: AS-PATH (предотвращение петель), NEXT-HOP, LOCAL-PREF, MED
- Выбор маршрута основан на политике, а не на технических метриках — в отличие от IGP
- BGP hijacking — один из главных рисков безопасности интернета

---

## Определения и термины

**BGP (Border Gateway Protocol)** — протокол внешней маршрутизации (EGP), обеспечивающий обмен маршрутной информацией между автономными системами.

**AS (Autonomous System, автономная система)** — совокупность маршрутизаторов под единым административным и техническим управлением, использующих один протокол внутренней маршрутизации.

**ASN (Autonomous System Number)** — глобально уникальный номер AS, назначаемый региональными регистраторами ICANN.

**Stub AS (тупиковая AS)** — AS, переносящая только свой трафик; часто не имеет собственного ASN.

**eBGP (external BGP)** — BGP-сессия между маршрутизаторами разных AS.

**iBGP (internal BGP)** — BGP-сессия между маршрутизаторами внутри одной AS.

**BGP-пир (BGP peer)** — маршрутизатор на конце BGP TCP-соединения. Сессия устанавливается только с явно сконфигурированными соседями.

**BGP Hijacking (захват префикса)** — инцидент, при котором AS объявляет о владении чужим IP-префиксом.

**Hot potato routing (горячая картошка)** — стратегия: AS стремится избавиться от пакета как можно быстрее, передав его в следующую AS через ближайший NEXT-HOP.

## Как устроено

### Автономные системы и структура интернета

Интернет — иерархическая структура из автономных систем:
- Каждая AS — отдельная организация (провайдер, университет, корпорация)
- Внутри AS используются IGP-протоколы (OSPF, RIP)
- Между AS используется BGP

### iBGP vs eBGP

| Аспект | eBGP | iBGP |
|---|---|---|
| Где работает | Между разными AS | Внутри одной AS |
| Назначение | Получение информации о достижимости подсетей от соседних AS | Распространение внешних маршрутов между всеми маршрутизаторами AS |
| TTL | 1 (обычно — прямое соединение) | Любое (маршрутизаторы не обязаны быть соседями физически) |

### BGP-сессии и TCP

- Пары маршрутизаторов обмениваются информацией по **полупостоянным TCP-соединениям** через **порт 179**
- Сессия устанавливается только с соседями, явно указанными администратором
- BGP не требует собственных механизмов надёжности — всё делает TCP

### Атрибуты BGP

| Атрибут | Описание | Роль |
|---|---|---|
| **AS-PATH** | Список номеров AS, через которые прошло объявление префикса | Предотвращение петель: если в списке есть номер своей AS — объявление отклоняется |
| **NEXT-HOP** | IP-адрес интерфейса маршрутизатора начала пути AS-PATH | Связывает внешнюю маршрутизацию с внутренней |
| **LOCAL-PREF** | Локальное предпочтение, устанавливается администратором внутри AS | Наивысший приоритет при выборе пути |
| **MED** | Multiple Exit Discriminator | Сообщает соседней AS о предпочтительном входном канале при нескольких соединениях |

### Процесс выбора маршрута (полный)

При нескольких путях к одному префиксу применяются правила по приоритету:

```
1. Наивысший LOCAL-PREF
       ↓ (при равенстве)
2. Кратчайший AS-PATH
       ↓ (при равенстве)
3. Предпочтение eBGP перед iBGP (метод «раннего выхода»)
       ↓ (при равенстве)
4. Наименьший MED (если маршруты от одной AS)
       ↓ (при равенстве)
5. Ближайший NEXT-HOP по IGP-метрике (hot potato routing)
       ↓ (при равенстве)
6. Дополнительные BGP-идентификаторы маршрутизатора
```

## Детали и нюансы

### BGP Hijacking

BGP основан на **доверии** между провайдерами — один из главных уязвимых элементов интернета.

**Механизм атаки:**
- AS объявляет чужой IP-префикс
- Если объявление содержит более специфичный префикс (например, /24 вместо /22) — маршрутизаторы по всему миру направят трафик к злоумышленнику
- Трафик может быть перехвачен или потерян

**Исторические инциденты:**
- **1997, AS7007** — ошибочное объявление привело к потере маршрутов во всём интернете
- **2008, Pakistan Telecom** — ошибка привела к ложному перенаправлению мирового трафика YouTube в их сеть

### Выборочное объявление маршрутов

Провайдер должен объявлять только свои сети и сети своих клиентов. Объявлять маршрут, полученный от провайдера А, провайдеру Б нельзя — иначе сеть станет бесплатным транзитом.

### Механика BGP Hijacking (детально)

Маршрутизаторы всегда выбирают маршрут по **longest matching prefix**. Злоумышленник анонсирует поддиапазон жертвы (например, `/17` вместо `/16`) → весь трафик к `/17` направляется злоумышленнику.

## Сравнения и trade-offs

### BGP vs IGP (OSPF, RIP)

| Аспект | BGP | IGP (OSPF, RIP) |
|---|---|---|
| Область применения | Между AS (EGP) | Внутри AS |
| Основа выбора пути | Политика, коммерческие соглашения | Технические метрики (задержка, стоимость) |
| Метрики | AS-PATH, LOCAL-PREF, MED | Хопы (RIP), стоимость (OSPF) |
| Масштаб | Сотни тысяч префиксов | Ограниченный масштаб одной AS |
| Транспорт | TCP (порт 179) | UDP/собственный транспорт |
| Сходимость | Медленная | Быстрая |

## Важно / подводные камни / best practices

1. **BGP — протокол доверия** — нет встроенной аутентификации источников маршрутов; BGP hijacking возможен из-за этого
2. **LOCAL-PREF имеет наивысший приоритет** — именно через него администратор управляет исходящим трафиком
3. **MED — для входящего трафика** — администратор может влиять, через какой линк входит трафик от соседней AS
4. **AS-PATH — единственная защита от петель** — маршрутизатор отклоняет объявление, содержащее свой ASN
5. **iBGP требует full mesh** — по умолчанию каждый маршрутизатор внутри AS должен иметь iBGP-сессию с каждым другим; для решения используются Route Reflectors
6. **Выборочное объявление** — никогда не объявлять маршруты одного провайдера другому, чтобы не стать бесплатным транзитом
7. **Longest prefix match** — механизм, делающий hijacking эффективным: более специфичный префикс всегда побеждает
