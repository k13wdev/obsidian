## Суть и контекст

Эталонные модели OSI и TCP/IP описывают, как организовано взаимодействие узлов в сети. OSI — теоретическая семиуровневая модель, TCP/IP — практическая четырёхуровневая реализация, лежащая в основе современного интернета.

---

## Ключевые идеи

- Стек протоколов — иерархически организованный набор протоколов, достаточный для организации взаимодействия узлов в сети
- Каждый уровень предоставляет службы вышележащим уровням, скрывая детали реализации
- OSI — 7 уровней: физический, канальный, сетевой, транспортный, сеансовый, представления, прикладной
- TCP/IP — 4 уровня: канальный, межсетевой, транспортный, прикладной
- Прикладной уровень TCP/IP объединяет сеансовый, представления и прикладной уровни OSI
- Передача данных основана на инкапсуляции: каждый уровень добавляет свой заголовок
- На принимающей стороне происходит декапсуляция: каждый уровень снимает свой заголовок

---

## Определения и термины

**Стек протоколов** — иерархически организованный набор протоколов, достаточный для организации взаимодействия узлов в сети.

**PDU (Protocol Data Unit)** — единица данных протокола на каждом уровне: бит, кадр, пакет, сегмент, сообщение.

**Инкапсуляция** — процесс добавления заголовка каждым нижележащим уровнем при отправке данных.

**Декапсуляция** — обратный процесс на принимающей стороне: каждый уровень снимает свой заголовок и передаёт payload вышестоящему уровню.

**Заголовок (Header)** — служебная информация, необходимая для правильной обработки пакета одноранговым уровнем на другом конце. Содержит адреса, контрольные суммы, порядковые номера и флаги.

**Интерсеть** — объединение различных сетей в единую транспортную систему (задача сетевого уровня).

**SDN (Software-Defined Networking)** — архитектура, в которой плоскость данных (программируемые коммутаторы) отделена от плоскости управления (контроллер). Решает проблему ручной настройки маршрутизаторов.

**OpenFlow** — протокол обмена между контроллером SDN и коммутатором. Управляет таблицами продвижения по правилам «соответствие + действие». Канал работает поверх TCP.

**MPLS (MultiProtocol Label Switching)** — технология коммутации по 20-битным меткам, добавляемым между заголовками L2 и L3. Аппаратный поиск по точному индексу метки — быстрее IP-маршрутизации (LPM).

## Как устроено

### Модель OSI — 7 уровней

#### 7. Прикладной уровень (Application)

- **Функции:** предоставление интерфейса сетевых услуг непосредственно пользовательским приложениям (почта, веб-страницы, передача файлов)
- **Протоколы:** HTTP, FTP, SMTP, DNS, Telnet
- **PDU:** Сообщение (Message)

#### 6. Уровень представления (Presentation)

- **Функции:** обеспечение понятности данных для приложений — перекодировка, сжатие, шифрование
- **Протоколы:** SSL (обеспечивает шифрование для прикладных служб на этом уровне)
- **PDU:** Сообщение

#### 5. Сеансовый уровень (Session)

- **Функции:** управление диалогом между приложениями — установление, поддержание и синхронизация сеанса, контрольные точки для восстановления
- **PDU:** Сообщение

#### 4. Транспортный уровень (Transport)

- **Функции:** обеспечение сквозной (end-to-end) доставки сообщений между прикладными процессами с заданным уровнем надёжности; разбивка длинных сообщений на сегменты
- **Протоколы:** TCP (надёжный), UDP (ненадёжный)
- **PDU:** Сегмент (Segment) или дейтаграмма

#### 3. Сетевой уровень (Network)

- **Функции:** маршрутизация пакетов и объединение различных сетей в интерсеть; глобальная адресация и выбор путей передачи между удалёнными хостами
- **Протоколы:** IP (IPv4, IPv6), ICMP, RIP, OSPF, BGP
- **PDU:** Пакет (Packet) / дейтаграмма
- **Оборудование:** маршрутизаторы (шлюзы), коммутаторы 3-го уровня

#### 2. Канальный уровень (Data Link)

- **Функции:** надёжная передача данных между двумя устройствами в пределах одного канала связи; формирование кадров (фреймов), обнаружение и коррекция ошибок (CRC), управление потоком и доступом к среде (MAC)
- **Протоколы:** Ethernet (802.3), Wi-Fi (802.11), PPP
- **PDU:** Кадр / Фрейм (Frame)
- **Оборудование:** мосты, коммутаторы локальных сетей, NIC

#### 1. Физический уровень (Physical)

- **Функции:** передача необработанного потока битов по физическим каналам связи (кабелям, волокну, радиоволнам); определение способов кодирования битов в сигналы и параметров интерфейсов
- **Протоколы:** RS-232, стандарты соединителей и кабелей
- **PDU:** Бит (Bit)
- **Оборудование:** сетевые адаптеры, повторители, концентраторы (хабы), порты коммутаторов и маршрутизаторов

---

### Модель TCP/IP — 4 уровня

#### 4. Прикладной уровень (Application)
Содержит все высокоуровневые протоколы. Объединяет функции сеансового, представительного и прикладного уровней OSI.

#### 3. Транспортный уровень (Transport)
Обеспечивает связь между прикладными процессами на разных хостах. Протоколы: TCP и UDP.

#### 2. Межсетевой уровень (Internet)
Обеспечивает перемещение пакетов по всей составной сети. Основной протокол — IP.

#### 1. Канальный уровень (Link)
Описывает взаимодействие хоста с линией связи; упаковывает IP-пакеты в кадры нижележащей технологии (Ethernet, DSL и др.). Объединяет функции физического и канального уровней OSI.

---

### Инкапсуляция и декапсуляция

**Инкапсуляция (отправка):**
- Каждый нижележащий уровень добавляет к данным свой заголовок
- Сообщение прикладного уровня → вкладывается в сегмент транспортного → помещается в пакет сетевого → упаковывается в кадр канального → передаётся битами

**Декапсуляция (приём):**
- Данные движутся вверх по стеку
- Каждый уровень анализирует и удаляет свой заголовок
- Передаёт чистую полезную нагрузку (payload) вышестоящему уровню

Заголовки содержат: адреса, контрольные суммы, порядковые номера, флаги. Чем сложнее функции протокола — тем сложнее структура его заголовка.

---

### SDN и OpenFlow

- Коммутаторы SDN — «глупые» устройства без встроенной логики маршрутизации
- Контроллер видит всю топологию и рассылает готовые таблицы продвижения
- OpenFlow 1.1+ поддерживает многотабличный конвейер (GO TO table) для сокращения числа правил
- При отсутствии совпадения — пакет отправляется на виртуальный порт CONTROLLER (сообщение `packet_in`)

## Сравнения и trade-offs

### Соответствие уровней OSI и TCP/IP

| OSI | TCP/IP | Примечание |
|---|---|---|
| 7. Прикладной | 4. Прикладной | TCP/IP объединяет 3 верхних уровня OSI |
| 6. Представления | 4. Прикладной | ↑ |
| 5. Сеансовый | 4. Прикладной | ↑ |
| 4. Транспортный | 3. Транспортный | Функционально идентичны |
| 3. Сетевой | 2. Межсетевой | Функционально идентичны |
| 2. Канальный | 1. Канальный | TCP/IP: интерфейс с линией связи |
| 1. Физический | 1. Канальный | TCP/IP объединяет 2 нижних уровня OSI |

### OSI vs TCP/IP

| Аспект | OSI | TCP/IP |
|---|---|---|
| Количество уровней | 7 | 4 |
| Назначение | Теоретическая эталонная модель | Практическая реализация |
| Применение | Обучение, проектирование | Реальный интернет |
| Гибкость | Детальное разделение функций | Упрощённая структура |

## Важно / подводные камни / best practices

- OSI — эталон для описания и понимания сетей, но не используется напрямую в реальных реализациях
- TCP/IP — реальный стек протоколов современного интернета
- Граница между уровнями в TCP/IP менее чёткая, чем в OSI
- SSL/TLS в модели OSI относится к уровню представления, но в TCP/IP входит в прикладной уровень
- OSI: сеансовый и представительский уровни оказались бесполезны для большинства приложений — в TCP/IP они отсутствуют

Как уровни OSI соответствуют уровням TCP/IP?
