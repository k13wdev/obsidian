## Суть и контекст

Декомпозиция отношений — процесс разбиения переменной отношения на проекции в ходе нормализации. Корректная декомпозиция должна быть **без потерь** (lossless) и, по возможности, **с сохранением зависимостей**. Теорема Хита — формальное основание для декомпозиции без потерь.

---

## Ключевые идеи

- Декомпозиция — разбиение исходной переменной отношения на проекции, обратное соединение которых восстанавливает исходное отношение.
- **Теорема Хита**: если R{A, B, C} удовлетворяет ФЗ A → B, то R равна соединению проекций {A, B} и {A, C}.
- Декомпозиция без потерь гарантирует восстановление исходного отношения через естественное соединение проекций.
- Декомпозиция с сохранением зависимостей гарантирует, что ФЗ могут контролироваться в рамках отдельных проекций.
- Достижение одновременно НФБК и сохранения всех зависимостей **математически невозможно** в некоторых случаях.

---

## Определения и термины

**Функциональная зависимость (ФЗ)** — связь типа «многие к одному» между множествами атрибутов внутри переменной отношения; A → B означает, что для каждого значения A существует ровно одно связанное с ним значение B.

**Детерминант** — левая часть функциональной зависимости (A в A → B).

**Частичная зависимость** — функциональная зависимость неключевого атрибута от части составного потенциального ключа (нарушение 2НФ).

**Транзитивная зависимость** — зависимость вида A → B и B → C, где A — первичный ключ, а B не является потенциальным ключом (нарушение 3НФ).

**Многозначная зависимость (МЗЗ)** — обобщение ФЗ; A ↠ B означает, что множество значений B, соответствующих паре AC, зависит только от значения A (нарушение 4НФ).

**Зависимость соединения (ЗС)** — обобщение МЗЗ; отношение удовлетворяет ЗС ∗(A, B, …, Z), если оно равно соединению своих проекций по подмножествам атрибутов A, B, …, Z (нарушение 5НФ). Связь: ФЗ ⊂ МЗЗ ⊂ ЗС.

**Тривиальная ЗС** — зависимость соединения, одно из подмножеств атрибутов которой включает все атрибуты исходного отношения.

**Декомпозиция без потерь (lossless decomposition)** — разбиение исходной переменной отношения на множество проекций, обладающих свойством гарантированного восстановления исходной переменной отношения за счёт обратной операции естественного соединения этих проекций.

**Декомпозиция с сохранением зависимостей (dependency-preserving decomposition)** — вариант декомпозиции, при котором совместный эффект от реализации ограничений ФЗ для отдельных проекций строго эквивалентен реализации всех ограничений для исходного множества ФЗ.

**Теорема Хита (Heath's theorem)** — если переменная отношения R{A, B, C} удовлетворяет функциональной зависимости A → B, то R равна соединению её проекций по атрибутам {A, B} и {A, C}.

## Как устроено

### Декомпозиция на НФБК (алгоритм Фейгина)

1. Поиск функциональной зависимости X → Y, нарушающей НФБК.
2. Замена переменной отношения двумя проекциями: первой — по атрибутам X и Y, второй — по атрибутам X и всем оставшимся, кроме Y.
3. Повторение до тех пор, пока все переменные отношения не окажутся в НФБК.

### Алгоритм декомпозиции до 3НФ (с сохранением зависимостей)

1. Найти неприводимое покрытие I для заданного множества ФЗ S.
2. Инициализировать пустое множество проекций D.
3. Для каждой ФЗ X → Y из I добавить в D проекцию по атрибутам {X, Y}.
4. Сгруппировать проекции с одинаковыми детерминантами X.
5. Добавить в D проекцию по любому потенциальному ключу исходного отношения R, если ни одна из имеющихся проекций его не содержит.

### Теорема Фейгина (Fagin's theorem)

Переменная отношения R{A, B, C} равна соединению своих проекций {A, B} и {A, C} тогда и только тогда, когда в ней выполняется МЗЗ A ↠ B | C.

### Визуализация декомпозиции без потерь

В центре верхнего блока — исходная таблица S с атрибутами S#, STATUS, CITY. Стрелки ведут к двум проекциям: SST (S#, STATUS) и SC (S#, CITY). При обратном проходе стрелки от SST и SC сходятся в операции JOIN, которая полностью (без фантомных строк) восстанавливает исходную таблицу S — благодаря соединению по первичному ключу S#.

## Детали и нюансы

- **Фантомные кортежи (spurious tuples)**: декомпозиция на проекции, не имеющие общих атрибутов, образующих потенциальный ключ хотя бы для одной из них, приводит к генерации фиктивных кортежей при обратном соединении. Семантическая связь безвозвратно утеряна.
- **Невозможность одновременного достижения НФБК и сохранения зависимостей**: пример — R(S, J, T) с ФЗ {S, J} → T и T → J. Разбиение на {S, T} и {T, J} для НФБК приводит к потере возможности контролировать ФЗ {S, J} → T в рамках одной таблицы. Требуется выбор между полным устранением аномалий и эффективным контролем ФЗ.
- **Атомарные отношения** (по Риссанену): отношения с перекрывающимися составными ключами, которые нельзя декомпозировать на независимые проекции даже при наличии нарушений НФБК. Пример: два ключа {S, J} и {T, J} при ФЗ T → J.
- **n-декомпонуемость**: существуют отношения, которые нельзя разбить на 2 проекции без потерь, но можно на 3 и более. Классический пример — циклическая зависимость SPJ (Поставщик, Деталь, Проект).
- **Принцип ортогональности**: для двух переменных отношения A и B не должно существовать проекций Aᵢ и Bⱼ с перекрывающимися смысловыми значениями — один и тот же кортеж не должен удовлетворять предикатам разных таблиц.

## Сравнения и trade-offs

| Цель | Плюсы | Минусы | Ограничение |
|------|-------|--------|-------------|
| Декомпозиция до НФБК | Полное устранение аномалий обновления | Может потерять возможность контролировать некоторые ФЗ в рамках одной таблицы | Не всегда совместима с сохранением зависимостей |
| Декомпозиция с сохранением зависимостей | ФЗ контролируются автоматически в каждой проекции | Может не достигать НФБК | Не всегда совместима с НФБК |

## Примеры

### Декомпозиция SECOND{S#, STATUS, CITY} для устранения транзитивной зависимости

Транзитивная зависимость: S# → CITY и CITY → STATUS. Разбиваем на две проекции:

```sql
/* Проекция 1: Поставщик → Город */
CREATE TABLE SC (S# CHAR(5), CITY CHAR(20), PRIMARY KEY (S#));
/* Проекция 2: Город → Статус */
CREATE TABLE CS (CITY CHAR(20), STATUS INTEGER, PRIMARY KEY (CITY));
```

Исходное отношение SECOND восстанавливается через: `SC JOIN CS` по атрибуту CITY.

## Важно / подводные камни / best practices

- **Не декомпозируйте без общих ключевых атрибутов** — обратное соединение сгенерирует фантомные кортежи (spurious tuples), и исходная семантика будет безвозвратно утеряна.
- **Теорема Хита — основа корректной декомпозиции** — всегда проверяйте, что ФЗ A → B действительно выполняется перед разбиением R{A, B, C} на {A, B} и {A, C}.
- **Компромисс НФБК vs сохранение зависимостей** — математически невозможно достичь обоих целей одновременно для некоторых схем. Решение зависит от приоритетов.
