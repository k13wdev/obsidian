# Coverage

## Суть и контекст

Coverage (покрытие тестами) в Go — процент выражений (statements) в коде, которые выполняются в ходе прогона тестов. Встроен в стандартный инструментарий `go test`. Позволяет найти непокрытые ветки кода, отправить отчёт в CI/CD и визуализировать покрытие в браузере или IDE.

---

## Ключевые идеи

- `go test -cover` выводит процент покрытия выражений
- `-coverprofile=cover.out` сохраняет детальный отчёт в файл
- `-covermode` — три режима: `set`, `count`, `atomic`
- `go tool cover -html` — визуализация покрытия в браузере (зелёный/красный)
- `go tool cover -func` — текстовый вывод покрытия по функциям
- Без `-coverpkg=./...` пакеты без тестовых файлов не учитываются
- `-covermode=atomic` обязателен при параллельных тестах
- 100% coverage не является обязательной целью — `main()` и конфигурационный код не всегда нужно покрывать

---

## Определения и термины

**Test coverage (покрытие тестами)** — процент выражений (statements) в коде, которые выполняются в ходе прогона тестов. По умолчанию учитываются только пакеты с тестовыми файлами `_test.go`; функция `main` по умолчанию не учитывается.

**coverprofile** — локальный файл отчёта, в котором сохраняются детальные результаты прогона тестов для последующего форматирования или отправки в инструменты качества кода. Генерируется флагом `-coverprofile=output_filename`.

**covermode** — параметр, определяющий алгоритм вычисления покрытия. Принимает одно из трёх значений: `set`, `count`, `atomic`.

---

## Как устроено

### Режимы `-covermode`

| Режим | Что фиксирует | Когда использовать |
|-------|--------------|-------------------|
| `set` | Факт выполнения выражения (да/нет) | По умолчанию; простые сценарии |
| `count` | Сколько раз выражение было выполнено | Поиск редко покрытых участков кода |
| `atomic` | Как `count`, но с атомарными счётчиками | Параллельные тесты (`t.Parallel`) |

При режимах `count`/`atomic` HTML-отчёт окрашивает интенсивно-зелёным часто выполняемые строки, бледно-зелёным — редко выполняемые.

### Вывод `go tool cover -func`

```
/Users/.../service.go:10:    GetUser     100.0%
/Users/.../service.go:25:    CreateUser  75.0%
/Users/.../service.go:45:    DeleteUser  0.0%
total: (statements)                       70.0%
```

Столбцы: `путь:строка`, `имя функции`, `процент покрытия`. Последняя строка — итоговая статистика.

### HTML-отчёт

`go tool cover -html=cover.out` открывает файл в браузере:
- **Зелёным** — строки, выполненные в ходе тестов
- **Красным** — строки, не выполненные
- При `count`/`atomic` — различные оттенки зелёного в зависимости от частоты вызова

## Примеры

### Основные команды

```bash
# Вывод процента покрытия в консоль
go test -cover ./...

# Покрытие по всем пакетам (включая без тестов)
go test -cover -coverpkg=./... ./...

# Сохранить отчёт в файл
go test -coverprofile=cover.out ./...

# HTML-визуализация
go tool cover -html=cover.out

# Текстовый вывод по функциям
go tool cover -func=cover.out

# Режим с подсчётом количества вызовов
go test -covermode=count -coverprofile=cover.out ./...

# Обязательно для параллельных тестов
go test -covermode=atomic -coverprofile=cover.out ./...
```

### Исключение файлов из отчёта (например, main.go)

```bash
go test -coverprofile=cover.tmp.out ./...
cat cover.tmp.out | grep -v "main.go" > cover.out
go tool cover -html=cover.out
```

## Важно / подводные камни / best practices

**`-coverpkg=./...` обязателен при полном замере:** Без него пакеты без тестовых файлов не попадают в расчёт — процент покрытия будет завышен.

**`-covermode=atomic` при параллельных тестах:** Режимы `set`/`count` не thread-safe; при конкурентном доступе к счётчикам возможны коллизии. IDE GoLand использует `atomic` по умолчанию.

**100% coverage не цель:** Если `main.go` содержит только конфигурацию и запуск — покрывать его юнит-тестами нецелесообразно. Практикуется ручное исключение через `grep -v`.

**`-coverprofile` в CI/CD:** Сохранённый файл передаётся в инструменты контроля качества (например, Codecov, SonarQube) для централизованного отслеживания динамики покрытия.

**Функция `main()` по умолчанию не учитывается:** Это нормально; при необходимости покрыть её — использовать `-coverpkg`.
