# Переменные и Hoisting

## Суть и контекст

JavaScript компилируется за микросекунды до выполнения. В этот момент движок обрабатывает все объявления переменных и функций — это и порождает эффект **hoisting** (поднятия). Понимание этого механизма объясняет поведение `var`, `let`, `const` и функций до строки их объявления.

## Ключевые идеи

- JS компилируется до выполнения — все объявления обрабатываются на фазе компиляции
- `var a = 2` — два шага: объявление (компиляция) + присваивание (выполнение)
- **Function declaration** поднимается целиком (с телом). **Function expression** — только переменная (как `undefined`)
- Функции поднимаются **раньше** переменных; дублирующие `var` игнорируются
- `var` — function scope, поднимается с `undefined`
- `let` / `const` — block scope, в TDZ до строки объявления
- **TDZ** зависит от времени выполнения, а не от позиции кода

## Определения и термины

**Hoisting (Поднятие)** — эффект, при котором объявления переменных и функций концептуально «переезжают» в начало своей области видимости из-за фазы компиляции. Метафора: «яйцо (объявление) появляется раньше курицы (присваивания)».

**Фаза компиляции** — этап, на котором движок обрабатывает все объявления до выполнения кода. Компилятор просит Scope зарегистрировать каждый идентификатор.

**LHS-поиск** — поиск «куда записать» (Left Hand Side). Выполняется при присваивании значения переменной.

**Function scope** — область видимости ограничена функцией или глобальным объектом. Блоки `if`, `for`, `while` не создают новой области для `var`.

**Block scope** — область видимости ограничена блоком `{ ... }`. Используется `let` и `const`.

**TDZ (Temporal Dead Zone, Временная мёртвая зона)** — период от начала выполнения блока до строки инициализации `let`/`const`, в котором обращение к переменной выбрасывает `ReferenceError`.

**Function Declaration** — объявление вида `function foo() {}`. Поднимается целиком вместе с телом.

**Function Expression** — объявление вида `var foo = function() {}`. Поднимается только переменная (`undefined`), тело — нет.

**Named Function Expression (NFE)** — функциональное выражение с именем: `var f = function sayHi() {}`. Имя доступно только внутри функции — используется для надёжной рекурсии.

## Как устроено

### Фаза компиляции → фаза выполнения

```
var a = 2;
```

Движок разбивает на два шага:

```
// Фаза компиляции:
var a;       // Scope регистрирует идентификатор a

// Фаза выполнения:
a = 2;       // LHS-поиск: найти a и записать 2
```

### Приоритет подъёма: функции > переменные

```javascript
foo(); // 1 — функция уже поднята

var foo;                     // ← игнорируется (foo уже занята функцией)

function foo() {
  console.log(1);
}

foo = function() {           // ← присваивание происходит при выполнении
  console.log(2);
};
```

При дублировании — функции поднимаются первыми, дублирующие `var` игнорируются, но последующие **function declaration** переопределяют предыдущие.

### var — function scope + hoisting с undefined

```javascript
console.log(x); // undefined (не ReferenceError — var поднята)
var x = 5;
console.log(x); // 5

// Как видит движок:
// var x;           ← поднято с undefined
// console.log(x);  → undefined
// x = 5;
// console.log(x);  → 5
```

- Область видимости: функция или глобальный объект
- Блоки `if`, `for` — не создают новой области для `var`
- Повторное объявление в той же области — без ошибки

```javascript
var x = 1;
var x = 2; // OK, x = 2
```

### let — block scope + TDZ

```javascript
console.log(y); // ReferenceError: Cannot access 'y' before initialization
let y = 5;
```

- Область видимости: блок `{ ... }`
- Повторное объявление в том же блоке — `SyntaxError`
- До строки объявления — в **TDZ**

### const — block scope + TDZ + обязательная инициализация

```javascript
const Z;        // SyntaxError: Missing initializer
const Z = 10;   // OK
Z = 20;         // TypeError: Assignment to constant variable
```

- Обязательна инициализация при объявлении
- Нельзя переприсвоить (`=`)
- Содержимое объекта/массива **изменять можно**:

```javascript
const obj = { a: 1 };
obj.a = 2;       // OK — меняем содержимое
obj = {};        // TypeError — меняем ссылку
```

## Детали и нюансы

### TDZ: зависит от времени, не от позиции кода

```javascript
let x = 1;
if (true) {
  console.log(x); // ReferenceError!
  let x = 2;      // ← внутренний x в TDZ от { до этой строки
}
```

Несмотря на наличие внешнего `x = 1`, блок `if` захватывается внутренним `let x`. Переменная находится в TDZ от `{` до строки `let x = 2`.

### TDZ и typeof

```javascript
console.log(typeof undeclaredVar); // "undefined" — безопасно
console.log(typeof tdzVar);        // ReferenceError — опасно!
let tdzVar = 1;
```

Обычно `typeof` безопасен для необъявленных переменных. Но если переменная в TDZ — `typeof` **тоже** выбросит `ReferenceError`.

### TDZ зависит от времени, а не от позиции

```javascript
function useLet() {
  console.log(x); // OK! — функция вызвана ПОСЛЕ объявления x
}
let x = 1;
useLet(); // 1
```

Функция определена до `let x`, но вызвана после — ошибки нет.

### let в циклах for: перепривязка на каждой итерации

```javascript
// var — одна переменная на все итерации
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Выведет: 3, 3, 3

// let — новая переменная для каждой итерации
for (let j = 0; j < 3; j++) {
  setTimeout(() => console.log(j), 100);
}
// Выведет: 0, 1, 2
```

`let` в `for` создаёт новую лексическую область видимости (и новую переменную) для каждой итерации. Каждое замыкание в `setTimeout` захватывает свою версию `j`.

### Function Expression vs Function Declaration

```javascript
// Function Expression: поднимается только переменная (undefined)
foo(); // TypeError: foo is not a function
var foo = function() { console.log('hi'); };

// Function Declaration: поднимается целиком
bar(); // "hi"
function bar() { console.log('hi'); }
```

## Сравнения и trade-offs

| | `var` | `let` | `const` |
|---|---|---|---|
| Scope | Function / Global | Block | Block |
| Hoisting | Да, с `undefined` | Да, но в TDZ | Да, но в TDZ |
| TDZ | Нет | Да | Да |
| Повторное объявление | Да | Нет (SyntaxError) | Нет (SyntaxError) |
| Переприсваивание | Да | Да | Нет (TypeError) |
| Обязательная инициализация | Нет | Нет | Да |
| Добавляется в `window` (глобал) | Да | Нет | Нет |

| | Function Declaration | Function Expression | Arrow Function |
|---|---|---|---|
| Hoisting | Целиком (имя + тело) | Только переменная (`undefined`) | Только переменная (`undefined`) |
| Вызов до объявления | Да | TypeError | TypeError |

## Примеры

### Пример: порядок поднятия

```javascript
console.log(a); // undefined (var поднята)
console.log(b); // ReferenceError (let в TDZ)

var a = 1;
let b = 2;
```

### Пример: функция переопределяет функцию при дублировании

```javascript
foo(); // "2" — последнее объявление функции выигрывает

function foo() { console.log("1"); }
function foo() { console.log("2"); }
```

### Пример: NFE для надёжной рекурсии

```javascript
var factorial = function fact(n) {
  return n <= 1 ? 1 : n * fact(n - 1); // fact — надёжная ссылка
};

var backup = factorial;
factorial = null;
backup(5); // 120 — fact всё ещё работает
```

## Важно / подводные камни / best practices

- **Всегда используй `let`/`const`** вместо `var` — предсказуемый block scope
- **`const` по умолчанию** — переключайся на `let` только если знаешь, что будешь переприсваивать
- **Не полагайся на hoisting** — объявляй переменные и функции до их использования
- **`var` в цикле + замыкание** — классический баг: используй `let` или IIFE
- **`typeof` не безопасен** для переменных в TDZ — не используй как проверку существования `let`/`const`
- **Не переопределяй `Function.prototype`** — приведёт к непредсказуемому hoisting

## Карточки для повторения

#flashcards

Что такое hoisting?::Эффект, при котором все объявления переменных и функций концептуально «переезжают» в начало области видимости из-за того, что движок обрабатывает их на фазе компиляции до выполнения кода.

Как движок обрабатывает `var a = 2`?::Два шага: 1) Фаза компиляции — `var a` — Scope регистрирует идентификатор. 2) Фаза выполнения — `a = 2` — LHS-поиск и присваивание.

Что происходит при hoisting function declaration?::Функция поднимается **целиком** — и имя, и тело. Её можно вызвать до строки объявления.

Что происходит при hoisting function expression (`var f = function() {}`)?::Поднимается только переменная `f` со значением `undefined`. Тело функции НЕ поднимается. Вызов до присваивания даст `TypeError`.

Кто поднимается первым — функции или переменные?::Функции. Дублирующие `var` с тем же именем игнорируются.

Что такое TDZ (Temporal Dead Zone)?::Период от начала выполнения блока до строки инициализации `let`/`const`. Любое обращение к переменной в TDZ выбрасывает `ReferenceError`.

Безопасен ли `typeof` для переменных в TDZ?::Нет! `typeof tdzVar` выбросит `ReferenceError`. Обычно `typeof` безопасен для необъявленных переменных — но не для тех, что в TDZ.

Какой scope у `var`?::Function scope (или глобальный). Блоки `if`, `for`, `while` не создают новой области видимости для `var`.

Какой scope у `let` и `const`?::Block scope — ограничен блоком `{ ... }`.

Можно ли переприсвоить `const`?::Нет — `TypeError`. Но содержимое объекта или массива изменять можно: `const arr = []; arr.push(1)` — OK.

Зачем `let` в `for` решает проблему замыканий?::`let` в заголовке `for` создаёт новую переменную для каждой итерации. Каждое замыкание захватывает свою версию переменной.

Что выведет код: `for (var i=0; i<3; i++) setTimeout(()=>console.log(i), 0)`?::3, 3, 3 — потому что `var i` общая для всех итераций, и к моменту срабатывания таймеров цикл завершился.

Что такое Named Function Expression (NFE) и зачем нужно?::Функциональное выражение с именем: `var f = function fact(n) {}`. Имя доступно только внутри функции и используется для надёжного рекурсивного вызова, даже если внешняя переменная была перезаписана.
