## Суть и контекст

Прототипное наследование — фундамент JavaScript. В отличие от классических ООП-языков (Java, C++), здесь нет копирования поведения от класса к экземпляру. Вместо этого объекты **связываются** друг с другом через ссылки `[[Prototype]]` и **делегируют** вызовы вверх по цепочке. Синтаксис `class` (ES6) — это лишь синтаксический сахар над этим механизмом.

## Ключевые идеи

- Каждый объект имеет скрытое свойство **`[[Prototype]]`** — ссылку на другой объект или `null`
- Поиск свойства идёт по **цепочке прототипов** вверх до `Object.prototype` → `null`
- `__proto__` — исторический геттер/сеттер `[[Prototype]]`; лучше использовать `Object.getPrototypeOf`
- Свойство `.prototype` у функций — **не** то же самое, что `[[Prototype]]` объекта
- `new Foo()` устанавливает `[[Prototype]]` экземпляра = `Foo.prototype`
- `Object.create(proto)` — создаёт объект с явным прототипом без конструктора
- `class` / `extends` / `super` — синтаксический сахар над прототипами
- Изменять `[[Prototype]]` существующего объекта крайне не рекомендуется — ломает оптимизации
- **Shadowing**: свойство в самом объекте скрывает одноимённое свойство в прототипе

## Определения и термины

**`[[Prototype]]`** — внутреннее скрытое свойство объекта. Ссылка на другой объект (его прототип) или `null`.

**Цепочка прототипов (Prototype Chain)** — последовательность объектов, связанных через `[[Prototype]]`. При поиске свойства движок проходит по ней вверх.

**`__proto__`** — исторический геттер/сеттер `[[Prototype]]` из `Object.prototype`. Стандартизирован в ES6 для совместимости. Рекомендуется использовать `Object.getPrototypeOf/setPrototypeOf`.

**`.prototype`** — публичное свойство **функции** (не объекта). При `new Foo()` становится `[[Prototype]]` нового экземпляра.

**`constructor`** — свойство объекта `Foo.prototype`, указывающее обратно на функцию `Foo`. По умолчанию неперечислимое. Ненадёжно — может быть перезаписано.

**`Object.create(proto)`** — создаёт новый объект с указанным прототипом (`[[Prototype]] = proto`). Позволяет настраивать делегирование без `new`.

**Shadowing (Затенение)** — свойство объекта, скрывающее (затеняющее) одноимённое свойство в цепочке прототипов.

**`instanceof`** — проверяет, присутствует ли `Constructor.prototype` где-либо в `[[Prototype]]`-цепочке объекта.

**`[[HomeObject]]`** — скрытое свойство метода, хранящее объект, в котором метод был объявлен. Используется `super` для корректного поиска.

**Делегирование (Delegation)** — паттерн, при котором объект не копирует поведение, а делегирует вызов методов объекту-прототипу через цепочку `[[Prototype]]`.

## Как устроено

### Цепочка прототипов и поиск свойства

```javascript
var animal = { eats: true };
var rabbit = Object.create(animal); // rabbit.[[Prototype]] = animal
rabbit.jumps = true;

// Поиск свойства rabbit.eats:
// 1. В rabbit — нет
// 2. В rabbit.[[Prototype]] = animal — есть! eats = true
console.log(rabbit.eats);  // true
console.log(rabbit.jumps); // true (есть в самом rabbit)

// Вершина цепочки:
// rabbit → animal → Object.prototype → null
```

Если свойство не найдено до `null` → возвращается `undefined`.

### `[[Prototype]]` vs `.prototype`

```javascript
function Foo() {}

// .prototype — свойство ФУНКЦИИ
Foo.prototype; // { constructor: Foo }

// Создание экземпляра через new
var obj = new Foo();

// [[Prototype]] экземпляра = Foo.prototype
Object.getPrototypeOf(obj) === Foo.prototype; // true
obj.__proto__ === Foo.prototype;              // true
```

**Что делает `new`:**
1. Создаёт новый пустой объект `{}`
2. Устанавливает `[[Prototype]]` нового объекта = `Foo.prototype`
3. Выполняет `Foo` с `this` = новый объект
4. Если `Foo` не возвращает объект явно — возвращает новый объект

### Свойство `constructor` — ненадёжно

```javascript
function Foo() {}
// По умолчанию: Foo.prototype.constructor === Foo

var a = new Foo();
a.constructor === Foo; // true — НО! a.constructor делегируется в Foo.prototype
                       // Это не "собственное" свойство a

// Если перезаписать прототип:
Foo.prototype = {};    // новый объект без constructor
var b = new Foo();
b.constructor === Foo; // false! b.constructor === Object
```

### Object.create

```javascript
// Создание объекта с явным прототипом
var person = {
  greet: function() { return 'Hi, I am ' + this.name; }
};

var vasya = Object.create(person);
vasya.name = 'Вася';
vasya.greet(); // 'Hi, I am Вася' — метод делегирован в person

// Object.create(null) — чистый словарь
var dict = Object.create(null);
dict.key = 'value';
// У dict нет toString, hasOwnProperty и т.д.
Object.hasOwn(dict, 'key'); // true (безопасно для null-прото)
```

### Затенение свойств (Shadowing)

Присваивание `obj.prop = value` записывает в **сам** `obj`, скрывая прототипное свойство. Но есть нюансы:

```javascript
// Нюанс 1: writable: false в прототипе
var proto = {};
Object.defineProperty(proto, 'x', { value: 1, writable: false });
var obj = Object.create(proto);
obj.x = 2; // В non-strict — молча игнорируется; В strict — TypeError
            // Затеняющее свойство НЕ создаётся!

// Нюанс 2: сеттер в прототипе
var proto2 = {
  get x() { return this._x; },
  set x(v) { this._x = v * 2; }  // ← вызовется сеттер, а не создастся свойство
};
var obj2 = Object.create(proto2);
obj2.x = 5; // Вызван сеттер: obj2._x = 10
```

### `instanceof` и цепочка прототипов

```javascript
function Animal() {}
function Rabbit() {}
Rabbit.prototype = Object.create(Animal.prototype);

var rabbit = new Rabbit();
rabbit instanceof Rabbit; // true — Rabbit.prototype есть в цепочке
rabbit instanceof Animal; // true — Animal.prototype есть в цепочке
rabbit instanceof Object; // true — Object.prototype есть в цепочке
```

**Ненадёжность `instanceof`:** если изменить `Constructor.prototype` после создания объекта или сравнивать объекты из разных iframe (разные конструкторы) — результат будет неверным.

### `class`, `extends`, `super`

```javascript
class Animal {
  constructor(name) {
    this.name = name;
  }
  walk() { console.log(`${this.name} идёт`); }
}

class Rabbit extends Animal {
  constructor(name) {
    super(name); // ОБЯЗАТЕЛЬНО вызвать super() до обращения к this
  }
  walk() {
    super.walk();              // Animal.prototype.walk через [[HomeObject]]
    console.log('и прыгает');
  }
}

var r = new Rabbit('Кролик');
r.walk(); // 'Кролик идёт' / 'и прыгает'
```

`extends` устанавливает **две** ссылки `[[Prototype]]`:
1. `Rabbit.prototype.[[Prototype]]` = `Animal.prototype` (наследование методов экземпляра)
2. `Rabbit.[[Prototype]]` = `Animal` (наследование статических методов)

**`super` привязан статически через `[[HomeObject]]`:**
```javascript
let animal = {
  walk() { console.log('animal walk'); }
};
let rabbit = {
  __proto__: animal,
  walk() {
    super.walk(); // [[HomeObject]] = rabbit → ищет в rabbit.[[Prototype]] = animal
  }
};

// Если "одолжить" метод:
let cat = {
  __proto__: animal,
  walk: rabbit.walk // ОПАСНО! [[HomeObject]] остаётся rabbit
};
cat.walk(); // Вызовет animal.walk (через rabbit.[[HomeObject]]), а не cat.[[Prototype]]
```

### Эквивалент class на прототипах

```javascript
// ES6 class:
class Animal {
  walk() { console.log('I walk'); }
}
class Rabbit extends Animal {
  walk() {
    super.walk();
    console.log('and jump');
  }
}

// Эквивалент «под капотом»:
function Animal() {}
Animal.prototype.walk = function() { console.log('I walk'); };

function Rabbit() {}
Rabbit.prototype = Object.create(Animal.prototype); // цепочка прото
Rabbit.prototype.constructor = Rabbit;              // восстанавливаем constructor

Rabbit.prototype.walk = function() {
  Animal.prototype.walk.call(this); // эмуляция super
  console.log('and jump');
};
```

## Детали и нюансы

### `__proto__` vs `Object.getPrototypeOf`

```javascript
// Не рекомендуется (историческое):
obj.__proto__ = proto;

// Правильно (ES5+):
Object.getPrototypeOf(obj);    // читать [[Prototype]]
Object.setPrototypeOf(obj, proto); // менять [[Prototype]]
```

### Изменение `[[Prototype]]` — запрещено на практике

Динамическое изменение `[[Prototype]]` у существующего объекта (через `__proto__` или `setPrototypeOf`) **ломает оптимизации движков** по доступу к свойствам и **сильно снижает производительность**. Лучше создать новый объект с нужным прототипом.

### `for...in` vs `Object.keys`

```javascript
var proto = { inherited: true };
var obj = Object.create(proto);
obj.own = 'yes';

// for...in — собственные + перечисляемые из цепочки
for (let key in obj) { console.log(key); } // 'own', 'inherited'

// Object.keys — только собственные перечисляемые
Object.keys(obj); // ['own']

// Фильтрация в for...in:
for (let key in obj) {
  if (Object.hasOwn(obj, key)) { /* только собственные */ }
}
```

## Сравнения и trade-offs

| Способ создания | Явный прототип | Использование |
|---|---|---|
| `{}` / литерал | `Object.prototype` | Обычные объекты данных |
| `new Foo()` | `Foo.prototype` | Множество однотипных объектов |
| `Object.create(proto)` | `proto` | Явное делегирование без конструктора |
| `Object.create(null)` | `null` | Чистые словари без методов |

| | `class` синтаксис | Прototипы напрямую |
|---|---|---|
| Читаемость | Высокая | Низкая (много бойлерплейта) |
| `extends` | Встроен | `Rabbit.prototype = Object.create(Animal.prototype)` |
| `super` | Автоматически через `[[HomeObject]]` | `Animal.prototype.walk.call(this)` вручную |
| Строгий режим | Методы класса всегда в strict | Нет |
| Под капотом | Те же прототипы | Те же прототипы |

## Примеры

### Цепочка прототипов вручную

```javascript
var base = { type: 'base' };
var middle = Object.create(base);
middle.level = 'middle';
var child = Object.create(middle);
child.own = 'own';

child.own;   // 'own' — в child
child.level; // 'middle' — делегировано в middle
child.type;  // 'base' — делегировано в base
child.foo;   // undefined — не найдено до null

// Цепочка: child → middle → base → Object.prototype → null
```

### Проверка собственности и instanceof

```javascript
class Shape {}
class Circle extends Shape {}

var c = new Circle();
c instanceof Circle; // true
c instanceof Shape;  // true
c instanceof Object; // true

Object.hasOwn(c, 'radius');             // false (нет такого свойства)
Circle.prototype.isPrototypeOf(c);      // true (альтернатива instanceof)
```

## Важно / подводные камни / best practices

- **`constructor` ненадёжен** — при перезаписи `Foo.prototype` он теряется; явно восстанавливай: `Rabbit.prototype.constructor = Rabbit`
- **Не меняй `[[Prototype]]` существующего объекта** — серьёзная деградация производительности
- **`super` в заимствованных методах** — `[[HomeObject]]` статичен; при копировании метода в другой объект `super` останется привязан к исходному
- **`Object.create(null)`** — используй для словарей-хэшмапов, чтобы избежать конфликтов с методами прототипа
- **Предпочитай `Object.getPrototypeOf`** над `__proto__`
- **`instanceof` ненадёжен** при изменении `prototype` или при проверке объектов из других iframe
- **`for...in` перебирает прототипы** — всегда фильтруй через `Object.hasOwn`, если нужны только собственные свойства
- **В классах `super()` обязателен** в `constructor` дочернего класса — до обращения к `this`

## Карточки для повторения

#flashcards/javascript/prototype

Что такое `[[Prototype]]`?::Внутреннее скрытое свойство каждого объекта. Ссылка на другой объект (прототип) или `null`. Используется для делегирования при поиске свойств.

Как работает поиск свойства по цепочке прототипов?::1. Движок ищет свойство в самом объекте. 2. Если нет — в `[[Prototype]]`. 3. Поднимается по цепочке до `Object.prototype`. 4. Если не найдено — возвращает `undefined`.

Чем `.prototype` у функции отличается от `[[Prototype]]` объекта?::`.prototype` — публичное свойство функции-конструктора. При `new Foo()` оно становится `[[Prototype]]` нового экземпляра. `[[Prototype]]` — внутренняя ссылка любого объекта на его прототип.

Что делает `new Foo()`?::1. Создаёт новый пустой объект. 2. Устанавливает его `[[Prototype]]` = `Foo.prototype`. 3. Выполняет `Foo` с `this` = новый объект. 4. Возвращает объект.

Почему `constructor` ненадёжен?::Это свойство объекта `Foo.prototype`, которое легко теряется при перезаписи прототипа. Экземпляр получает его через делегирование, а не как собственное свойство.

Что делает `Object.create(proto)`?::Создаёт новый объект с `[[Prototype]] = proto`. Позволяет настраивать делегирование без использования конструкторов.

Что такое shadowing (затенение) в прототипах?::Свойство в самом объекте скрывает одноимённое свойство из цепочки прототипов. При чтении используется свойство объекта, а не прототипа.

Когда затенение свойства НЕ работает при присваивании?::Если в прототипе свойство: 1) `writable: false` — запись игнорируется. 2) Имеет сеттер — вызывается сеттер, новое свойство не создаётся.

Что такое `instanceof`?::Проверяет, присутствует ли `Constructor.prototype` где-либо в `[[Prototype]]`-цепочке объекта. Ненадёжен при изменении прототипа или сравнении из разных iframe.

Что делает `extends` под капотом?::Устанавливает две `[[Prototype]]` ссылки: 1. `Child.prototype.[[Prototype]] = Parent.prototype` (методы экземпляра). 2. `Child.[[Prototype]] = Parent` (статические методы).

Что такое `[[HomeObject]]`?::Скрытое свойство метода, хранящее объект, в котором метод был объявлен. `super` использует `[[HomeObject]]` для поиска родительского прототипа. Назначается статически.

Почему нельзя динамически менять `[[Prototype]]` существующего объекта?::Это ломает внутренние оптимизации движка по доступу к свойствам и сильно снижает производительность. Лучше создать новый объект с нужным прототипом.

Чем `Object.create(null)` полезен?::Создаёт объект без прототипа — «чистый словарь». У него нет `toString`, `hasOwnProperty` и других методов от `Object.prototype`. Безопасен для хранения произвольных ключей.

Что выведет `for...in` vs `Object.keys` для объекта с прототипом?::`for...in` — собственные + перечисляемые наследованные свойства. `Object.keys` — только собственные перечисляемые.

Нужно ли вызывать `super()` в конструкторе дочернего класса?::Да, обязательно и до первого обращения к `this`. Иначе — ReferenceError.
