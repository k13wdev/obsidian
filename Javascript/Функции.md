## Суть и контекст

Функции в JavaScript являются **объектами первого класса (first-class citizens)**: их можно передавать как аргументы, возвращать из других функций и присваивать переменным. На этом строятся паттерны высшего порядка, замыкания и декораторы.

## Ключевые идеи

- Функции — это «вызываемые объекты» (со скрытым свойством `[[Call]]`)
- Три способа создания: **declaration**, **expression**, **arrow**
- Function declaration **поднимается** целиком; expression — только переменная
- Arrow function **не имеет** собственного `this`, `arguments`, `super`, `new.target`
- Arrow function **не может** быть конструктором (`new`) и не имеет `prototype`
- `arguments` — псевдомассив, синхронизирован с параметрами в non-strict
- `rest` (`...args`) — настоящий массив оставшихся аргументов
- Значения по умолчанию вычисляются **лениво** (только если аргумент `undefined`)
- **IIFE** — создаёт новую область видимости для инкапсуляции
- **Рекурсия** — каждый вызов создаёт новый LexicalEnvironment; TCO в ES6 strict mode

## Определения и термины

**First-class function (Функция первого класса)** — функция, которую можно присвоить переменной, передать как аргумент и вернуть из другой функции. В JS функция является «вызываемым объектом» (`[[Call]]`).

**Higher-order function (Функция высшего порядка)** — функция, принимающая другую функцию как аргумент или возвращающая функцию.

**Decorator (Декоратор)** — функция-обёртка над другой функцией. Расширяет её поведение (логирование, кэширование), прозрачно передавая `this` и аргументы через `func.apply(this, arguments)`.

**IIFE (Immediately Invoked Function Expression)** — функциональное выражение, немедленно вызываемое после объявления: `(function() { ... })()`. Создаёт изолированную область видимости.

**Named Function Expression (NFE)** — функциональное выражение с именем: `var f = function name() {}`. Имя доступно только внутри тела функции — используется для надёжной рекурсии.

**arguments** — псевдомассив (array-like), доступный внутри обычных функций. Содержит все переданные аргументы, имеет `length` и индексы, но не имеет методов массива (`forEach`, `map`).

**Rest-параметры (`...args`)** — синтаксис ES6: собирает все оставшиеся аргументы в настоящий массив. Должен быть последним параметром.

**Tail Call Optimization (TCO)** — оптимизация хвостового вызова в ES6 strict mode: если рекурсивный вызов стоит в хвостовой позиции (`return foo(x)`), движок переиспользует текущий стек-фрейм вместо создания нового.

**Стек вызовов (Call Stack)** — структура данных, хранящая контексты выполнения активных функций. При каждом вызове новый контекст добавляется; при завершении — удаляется.

## Как устроено

### Function Declaration vs Function Expression vs Arrow

**Function Declaration:**
```javascript
function greet(name) {
  return 'Hello, ' + name;
}

greet('Вася'); // 'Hello, Вася'
// Можно вызвать ДО объявления — поднимается целиком
```

**Function Expression:**
```javascript
var greet = function(name) {
  return 'Hello, ' + name;
};
// Нельзя вызвать ДО строки присваивания — TypeError
```

**Named Function Expression (NFE):**
```javascript
var factorial = function fact(n) {
  return n <= 1 ? 1 : n * fact(n - 1); // fact — надёжная ссылка внутри
};
// fact — недоступен снаружи
```

**Arrow Function:**
```javascript
const greet = (name) => 'Hello, ' + name;

// Без скобок — один параметр
const double = x => x * 2;

// С телом — нужен explicit return
const sum = (a, b) => {
  const result = a + b;
  return result;
};
```

### Arrow Function: ключевые отличия

```javascript
const obj = {
  name: 'Вася',
  // Обычный метод — this = obj при вызове obj.greet()
  greet: function() {
    setTimeout(function() {
      console.log(this.name); // undefined — this потерян
    }, 100);
  },
  // С arrow — this берётся из greet (лексически)
  greetArrow: function() {
    setTimeout(() => {
      console.log(this.name); // 'Вася' — this из внешней функции
    }, 100);
  }
};
```

Arrow function **нельзя использовать** как:
- Конструктор (`new Arrow()` → `TypeError`)
- Метод с динамическим `this`
- Функцию с `arguments` (используй `...rest`)

### arguments vs rest

```javascript
// arguments — псевдомассив, в обычных функциях
function sum() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}

// Конвертация в массив (старый способ):
var args = [].slice.call(arguments);

// rest — настоящий массив (ES6)
function sumRest(...nums) {
  return nums.reduce((acc, n) => acc + n, 0);
}
sumRest(1, 2, 3); // 6
```

**Синхронизация arguments с параметрами (non-strict):**
```javascript
function foo(x) {
  arguments[0] = 10;
  console.log(x); // 10 — синхронизировано!
}
foo(1);
// В ES6 при использовании default/rest — синхронизация отключается
```

### Параметры по умолчанию (Default parameters)

```javascript
// Старый способ — баг при 0 или ""
function setWidth(width) {
  width = width || 200; // ПЛОХО: 0 || 200 = 200
}

// ES6 — ленивое вычисление
function setWidth(width = 200) { ... }  // OK: 0 → 0, undefined → 200

// Можно использовать выражение или вызов функции
function createUser(name, role = getDefaultRole()) {
  // getDefaultRole() вызывается ТОЛЬКО если role не передан
}
```

### IIFE

```javascript
// Классический синтаксис
(function() {
  var private = 'скрытая переменная';
  // private не попадает в глобальный scope
})();

// ES6 стрелочный IIFE
(() => {
  const private = 'скрытая';
})();
```

Зачем: создать изолированную область видимости, скрыть переменные, создать модуль.

### Рекурсия и стек

```javascript
// Каждый вызов → новый контекст в стеке
function factorial(n) {
  if (n <= 1) return 1;
  return n * factorial(n - 1); // Не хвостовой вызов — TCO не применяется
}

// Хвостовой вызов — TCO применяется в strict mode ES6
function factTCO(n, acc = 1) {
  "use strict";
  if (n <= 1) return acc;
  return factTCO(n - 1, n * acc); // Хвостовой — текущий стек-фрейм переиспользуется
}
```

### Декоратор с call forwarding

```javascript
function makeLogging(f, logArray) {
  return function wrapper() {
    var args = [].slice.call(arguments);
    logArray.push(args);
    return f.apply(this, arguments); // прозрачная передача this и аргументов
  };
}

var user = {
  name: 'Вася',
  work: function(a, b) {
    console.log(this.name + ' вычисляет: ' + (a + b));
    return a + b;
  }
};

var log = [];
user.work = makeLogging(user.work, log);
user.work(1, 2); // 'Вася вычисляет: 3'
```

## Детали и нюансы

- Arrow function и `bind`/`call`/`apply` — вызов не изменит `this` стрелочной функции
- `arguments` недоступен в arrow function — используй `...rest`
- NFE: имя функции недоступно **снаружи**, только внутри тела
- Параметры по умолчанию вычисляются при каждом вызове, а не один раз
- TCO поддерживается не всеми браузерами (несмотря на ES6)

## Сравнения и trade-offs

| | Function Declaration | Function Expression | Arrow Function |
|---|---|---|---|
| Hoisting | Целиком | Только переменная | Только переменная |
| `this` | Динамический | Динамический | Лексический (внешний) |
| `arguments` | Есть | Есть | Нет |
| `new` | Да | Да | Нет (TypeError) |
| `prototype` | Есть | Есть | Нет |
| Анонимность | Нет | Может быть | Всегда анонимна |

| | `arguments` | `rest (...args)` |
|---|---|---|
| Тип | Псевдомассив (array-like) | Настоящий массив |
| Методы массива | Нет | Да (`sort`, `map`, `forEach`) |
| Доступен в arrow | Нет | Да |
| Все аргументы | Да | Только оставшиеся |

## Примеры

### Частичное применение через bind

```javascript
function multiply(a, b) {
  return a * b;
}

const double = multiply.bind(null, 2); // фиксируем a = 2
double(5); // 10
double(7); // 14
```

### Higher-order function: map, filter

```javascript
const nums = [1, 2, 3, 4, 5];
const evens = nums.filter(n => n % 2 === 0); // [2, 4]
const doubled = nums.map(n => n * 2);         // [2, 4, 6, 8, 10]
```

## Важно / подводные камни / best practices

- **Prefer arrow functions** для коллбэков — избегаешь потери `this`
- **Не используй `arguments`** в новом коде — используй `...rest`
- **Default parameter `||`** — баг при `0` или `""`. Используй `= default` из ES6
- **IIFE** — используй для создания модулей и изоляции scope
- **NFE** — при рекурсии в function expression, а не обычная ссылка через переменную
- **Декоратор** — всегда используй `func.apply(this, arguments)` для прозрачного форвардинга
- **Стрелочная функция как метод объекта** — не делай так: `this` будет из внешней области, а не объекта

## Карточки для повторения

#flashcards/javascript/function

Что такое first-class function?::Функция, которую можно присвоить переменной, передать как аргумент и вернуть из другой функции. В JS функция — это «вызываемый объект» со свойством `[[Call]]`.

Чем arrow function отличается от обычной по части `this`?::Arrow function не имеет собственного `this`. Она использует лексический `this` — заимствует его из внешней (окружающей) области видимости в момент создания.

Можно ли изменить `this` стрелочной функции через `call`/`apply`/`bind`?::Нет. Лексическая привязка `this` стрелочной функции невозможно изменить никаким способом.

Можно ли вызвать arrow function как конструктор (`new`)?::Нет. Выбросит `TypeError`. У arrow function нет `prototype` и нет собственного `this`.

Чем `arguments` отличается от rest-параметров?::arguments — псевдомассив без методов массива, недоступен в arrow function. rest (`...args`) — настоящий массив с методами, содержит только оставшиеся аргументы.

Что такое IIFE и зачем нужен?::Immediately Invoked Function Expression — функция, выполняемая сразу после объявления: `(function(){})()`. Создаёт изолированную область видимости, предотвращает загрязнение глобального scope.

Что такое декоратор в JS?::Функция-обёртка над другой функцией. Расширяет поведение (логирование, кэширование), прозрачно передавая оригинальной функции `this` и аргументы через `func.apply(this, arguments)`.

Когда вычисляется значение по умолчанию параметра?::Лениво — только если аргумент не передан или передан как `undefined`. Вычисляется при каждом вызове функции.

Почему `width = width || 200` плохо?::Потому что `0 || 200 = 200`. При передаче `0` или `""` значение по умолчанию будет использоваться ошибочно. Лучше `width = 200` в сигнатуре ES6.

Что такое Named Function Expression и зачем оно?::Функциональное выражение с именем: `var f = function myName() {}`. Имя доступно только внутри функции — используется для надёжного рекурсивного вызова, даже если внешняя переменная перезаписана.

Что такое Tail Call Optimization (TCO)?::Оптимизация ES6 strict mode: если рекурсивный вызов стоит в хвостовой позиции (`return foo(x)`), движок переиспользует текущий стек-фрейм, предотвращая переполнение стека.

Есть ли `arguments` в стрелочной функции?::Нет. Используй `...rest` параметры.
