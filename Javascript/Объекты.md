## Суть и контекст

Объекты в JavaScript — коллекции свойств (пар «ключ-значение»), где ключи — строки или `Symbol`, а значения — любого типа. Это основной инструмент структурирования данных и поведения. Все сущности JS (массивы, функции, Date) — тоже объекты.

## Ключевые идеи

- Объекты — коллекции пар ключ-значение; ключи — строки или Symbol
- 4 способа создания: **литерал**, **конструктор + new**, **Object.create**, **class**
- Каждое свойство имеет **дескриптор** с флагами `writable`, `enumerable`, `configurable`
- Свойства могут быть **дата-свойствами** (value + writable) или **аксессорами** (get + set)
- `in` проверяет по цепочке прототипов; `hasOwnProperty` — только собственные
- `Object.freeze` > `Object.seal` > `Object.preventExtensions` — уровни неизменяемости
- Деструктуризация + rest (`...`) — удобная работа с полями объекта

## Определения и термины

**Объект** — коллекция свойств (пар «ключ-значение»), где ключи — строки или Symbol, значения — любого типа.

**Дескриптор свойства** — скрытый объект, описывающий характеристики свойства: `value`, `writable`, `enumerable`, `configurable`.

**Дата-свойство (Data property)** — обычное свойство с `value` и `writable`.

**Свойство-аксессор (Accessor property)** — свойство с `get` и `set` функциями вместо `value`/`writable`. При обращении вызывается геттер, при присваивании — сеттер.

**`writable`** — если `false`: значение нельзя изменить присваиванием (в strict mode → `TypeError`).

**`enumerable`** — если `false`: свойство не перечисляется в `for...in` и `Object.keys()`.

**`configurable`** — если `false`: свойство нельзя удалить (`delete`), изменить тип и большинство флагов дескриптора.

**`Object.freeze`** — высший уровень поверхностной неизменяемости: все свойства `writable: false` + `configurable: false` + запрет добавления новых.

**`Object.seal`** — все свойства `configurable: false` + запрет добавления новых (изменять существующие можно).

**`Object.preventExtensions`** — только запрет добавления новых свойств.

**Деструктуризация** — синтаксис для «распаковки» свойств объекта в отдельные переменные.

## Как устроено

### 4 способа создания объектов

**1. Объектный литерал** — самый распространённый:
```javascript
// ES5
var user = { name: 'Вася', isAdmin: true };

// ES6: короткие свойства + вычисляемые ключи
let name = 'Вася';
let user = {
  name,                           // shorthand: name: name
  isAdmin: true,
  ['user_' + name]: 'active'      // вычисляемый ключ
};
```

**2. Функция-конструктор + `new`:**
```javascript
function User(name) {
  this.name = name;
  this.sayHi = function() { alert(this.name); };
}

let vasya = new User('Вася');
// new: создаёт {}, привязывает к this, выполняет тело, возвращает объект
```

**3. `Object.create(proto)`:**
```javascript
const person = { isHuman: false };
const me = Object.create(person); // me.[[Prototype]] = person
me.name = 'Matthew';
console.log(me.isHuman); // false (делегировано в person)

// Чистый словарь (без прототипа):
const dict = Object.create(null); // нет toString, hasOwnProperty и т.д.
```

**4. Класс (ES6):**
```javascript
class User {
  constructor(name) {
    this.name = name;        // свойство экземпляра
  }
  sayHi() {                  // метод записывается в User.prototype
    alert(this.name);
  }
}
let user = new User('Вася');
```

### Проверка существования свойств

При чтении несуществующего свойства — `undefined`. Но свойство может существовать и равняться `undefined`:

```javascript
let obj = { age: undefined };

// undefined — не отличает отсутствие от значения undefined
console.log(obj.age);   // undefined
console.log(obj.name);  // undefined

// in — проверяет по цепочке прототипов
console.log('age' in obj);  // true
console.log('name' in obj); // false

// hasOwnProperty — только собственные свойства
obj.hasOwnProperty('age');  // true

// Object.hasOwn (современный, работает даже для Object.create(null))
Object.hasOwn(obj, 'age'); // true
```

### Дескрипторы свойств

По умолчанию при создании через литерал — все флаги `true`. При `Object.defineProperty` — все флаги `false` если не указаны явно.

```javascript
let user = {};

Object.defineProperty(user, 'name', {
  value: 'Вася',
  writable: false,      // нельзя изменить
  enumerable: true,     // видно в for...in и Object.keys
  configurable: false   // нельзя удалить или переконфигурировать
});

user.name = 'Петя'; // в strict — TypeError; в non-strict — молча игнорируется
delete user.name;   // false (не удалено)
```

### Геттеры и сеттеры

```javascript
let user = {
  firstName: 'Вася',
  surname: 'Петров',

  get fullName() {
    return `${this.firstName} ${this.surname}`;
  },

  set fullName(value) {
    [this.firstName, this.surname] = value.split(' ');
  }
};

console.log(user.fullName);   // 'Вася Петров' — вызван геттер
user.fullName = 'Петя Иванов'; // вызван сеттер
console.log(user.firstName);   // 'Петя'
```

Аксессоры через `Object.defineProperty`:
```javascript
Object.defineProperty(user, 'fullName', {
  get() { return `${this.firstName} ${this.surname}`; },
  set(value) { [this.firstName, this.surname] = value.split(' '); }
});
```

### Методы Object

```javascript
const user = { name: 'Вася', age: 30, _pass: 'secret' };

Object.keys(user);    // ['name', 'age', '_pass'] — собственные enumerable ключи
Object.values(user);  // ['Вася', 30, 'secret'] — значения
Object.entries(user); // [['name','Вася'], ['age',30], ['_pass','secret']] — пары

// Поверхностное клонирование
const clone = Object.assign({}, user);
// Merge нескольких объектов
const merged = Object.assign({}, defaults, overrides);

// Уровни неизменяемости (поверхностные)
Object.preventExtensions(user); // нельзя добавить новые свойства
Object.seal(user);              // + нельзя удалить/переконфигурировать
Object.freeze(user);            // + нельзя изменить значения
```

### Деструктуризация

```javascript
let options = { title: 'Меню', width: 100, height: 200 };

// Базовая
let { title, width, height } = options;

// Переименование
let { title, width: w, height: h } = options;

// Значение по умолчанию
let { title, width = 50, depth = 10 } = options; // depth = 10 (нет в options)

// Rest — оставшиеся свойства
let { title, ...size } = options;
// title = 'Меню', size = { width: 100, height: 200 }

// Деструктуризация в параметрах функции
function showMenu({ title = 'Заголовок', width = 100, height = 200 } = {}) {
  console.log(title, width, height);
}
showMenu({ title: 'Моё меню' }); // 'Моё меню', 100, 200
```

## Детали и нюансы

### Отличие `Object.freeze` от `Object.seal`

| | Новые свойства | Удаление свойств | Изменение значений |
|---|---|---|---|
| `preventExtensions` | ❌ | ✅ | ✅ |
| `seal` | ❌ | ❌ | ✅ |
| `freeze` | ❌ | ❌ | ❌ |

Все методы **поверхностные** (shallow) — вложенные объекты не замораживаются.

### configurable: false — одностороннее изменение

Если `configurable: false`, единственное разрешённое изменение — снизить `writable` с `true` до `false`. Обратно — нельзя.

### Object.keys vs for...in

```javascript
// Object.keys — только собственные enumerable свойства
Object.keys(obj); // ['a', 'b']

// for...in — собственные + наследованные enumerable свойства (по цепочке прототипов)
for (let key in obj) { ... }
// Фильтрация только собственных:
for (let key in obj) {
  if (obj.hasOwnProperty(key)) { ... }
}
```

### Object.assign — поверхностное копирование

```javascript
const original = { a: 1, nested: { b: 2 } };
const copy = Object.assign({}, original);

copy.a = 99;          // original.a = 1 (примитив — скопирован)
copy.nested.b = 99;   // original.nested.b = 99! (объект — скопирована ссылка)
```

## Сравнения и trade-offs

| Способ создания | Использование | Прототип |
|-----------------|--------------|---------|
| Литерал `{}` | Хранение данных, конфигурации | `Object.prototype` |
| `new Constructor` | Множество однотипных объектов | `Constructor.prototype` |
| `Object.create(proto)` | Делегирование без конструктора | Указанный `proto` |
| `class` | ООП-наследование | Класс-родитель |
| `Object.create(null)` | Чистый словарь без методов | `null` |

| Проверка свойства | Что проверяет |
|-------------------|--------------|
| `'key' in obj` | Собственные + прототипы |
| `obj.hasOwnProperty('key')` | Только собственные |
| `Object.hasOwn(obj, 'key')` | Только собственные (безопасно для `null`-прото) |
| `obj.key !== undefined` | Не отличает отсутствие от значения `undefined` |

## Примеры

### Вычисляемые свойства ES6

```javascript
const prefix = 'user';
let obj = {
  [`${prefix}_name`]: 'Вася',
  [`${prefix}_id`]: 42
};
// { user_name: 'Вася', user_id: 42 }
```

### Spread оператор для слияния объектов (ES2018)

```javascript
const defaults = { color: 'red', size: 'm' };
const custom = { size: 'xl', weight: 2 };
const result = { ...defaults, ...custom };
// { color: 'red', size: 'xl', weight: 2 }
```

### Object.defineProperty для read-only свойства

```javascript
const config = {};
Object.defineProperty(config, 'API_URL', {
  value: 'https://api.example.com',
  writable: false,
  configurable: false,
  enumerable: true
});
config.API_URL = 'другое'; // молча игнорируется (или TypeError в strict)
```

## Важно / подводные камни / best practices

- **`Object.assign` поверхностный** — вложенные объекты копируются по ссылке. Для глубокого клонирования — `structuredClone()` или `JSON.parse(JSON.stringify())`
- **`Object.freeze` поверхностный** — вложенные объекты не заморожены
- **`Object.create(null)`** — для словарей, когда не нужны методы прототипа (`toString`, `hasOwnProperty`)
- **Предпочитай `Object.hasOwn`** над `hasOwnProperty` — работает безопасно для любых объектов
- **Не переопределяй встроенные методы** (toString, valueOf) без крайней необходимости
- **Геттеры без сеттеров** — при попытке записи: молча игнорируется (non-strict) или `TypeError` (strict)
- **Деструктуризация с дефолтом `= {}`** в параметрах функции — защита от `undefined` аргумента

## Карточки для повторения

#flashcards/javascript/object

Перечисли 4 способа создания объектов в JavaScript::1. Объектный литерал `{}`. 2. Функция-конструктор + `new`. 3. `Object.create(proto)`. 4. Класс (`class`).

Чем `in` отличается от `hasOwnProperty`?::`in` проверяет наличие свойства в самом объекте **и** по цепочке прототипов. `hasOwnProperty` — только собственные свойства объекта, игнорируя прототипы.

Что такое дескриптор свойства?::Скрытый объект с характеристиками свойства: `value`, `writable`, `enumerable`, `configurable` (для дата-свойства) или `get`, `set`, `enumerable`, `configurable` (для аксессора).

Что означает флаг `writable: false`?::Значение свойства нельзя изменить присваиванием. В strict mode — `TypeError`. В non-strict — молча игнорируется.

Что означает флаг `enumerable: false`?::Свойство не будет видно в `for...in` и не попадёт в `Object.keys()`.

Что означает флаг `configurable: false`?::Свойство нельзя удалить (`delete`), изменить его тип и большинство флагов дескриптора (кроме снижения `writable: true → false`).

Какая разница между `Object.freeze`, `Object.seal`, `Object.preventExtensions`?::preventExtensions — нельзя добавить новые свойства. seal + нельзя удалить/переконфигурировать. freeze + нельзя изменить значения. Все поверхностные.

Что такое свойство-аксессор (accessor property)?::Свойство с функциями `get` и `set` вместо `value`/`writable`. При чтении вызывается `get`, при записи — `set`.

Что делает `Object.assign`?::Копирует все собственные enumerable свойства из объектов-источников в целевой объект. Копирование **поверхностное** — объекты копируются по ссылке.

Чем `Object.create(null)` отличается от `Object.create(Object.prototype)`?::При `null` — объект без прототипа, нет методов `toString`, `hasOwnProperty`. Чистый словарь. При `Object.prototype` — обычный объект с доступом к стандартным методам.

Как деструктурировать объект с переименованием и дефолтом?::`let { width: w = 100, height: h = 200 } = options;` — `w` получает значение `options.width` или `100`; `h` — `options.height` или `200`.

Как собрать оставшиеся свойства в destr rest?::`let { title, ...rest } = obj;` — `rest` получит все свойства кроме `title`.
