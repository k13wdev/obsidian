# Настройка WG-Easy (WireGuard + веб-интерфейс)

Продолжение после [vps-setup.md](vps-setup.md) и [docker-services-setup.md](docker-services-setup.md): сервер обновлён, пользователь **super**, SSH на порту **28472**, подключение через `ssh vps`, Docker и базовая структура `~/docker` уже подготовлены.

---

## Каталог и docker-compose

**Где:** на сервере под super

Скачать compose для WG-Easy в подкаталог **~/docker/wg-easy**:

```bash
mkdir -p ~/docker/wg-easy
curl -o ~/docker/wg-easy/docker-compose.yml https://raw.githubusercontent.com/wg-easy/wg-easy/master/docker-compose.yml
```

Образ в файле по умолчанию: `ghcr.io/wg-easy/wg-easy:15`. При желании зафиксировать конкретный минорный тег (например `15.0`) — отредактировать `image:` в `docker-compose.yml`.

---

## Запуск WG-Easy

**Где:** на сервере под super

```bash
cd ~/docker/wg-easy
docker-compose up -d
```

Проверка:

```bash
docker-compose ps
```

- **51820/UDP** — WireGuard (подключения клиентов).
- **51821/TCP** — веб-интерфейс (панель управления).

Использовать для запуска/остановки именно `docker-compose up -d` и `docker-compose down`, а не `start`/`stop`, иначе возможны проблемы при следующем старте.

## Доступ к веб-интерфейсу через SSH (проброс порта)

Панель не открывать в интернет. Подключаться только через туннель.

**Где:** локально

В `~/.ssh/config` в блок для **vps** добавить строку:

```
  LocalForward 51821 127.0.0.1:51821
```

Пример полного блока (если WG-Easy один):

```
Host vps
  HostName 195.62.49.129
  User super
  IdentitiesOnly yes
  IdentityFile ~/.ssh/id_rsa_vps
  Port 28472
  LocalForward 51821 127.0.0.1:51821
```

Если уже есть проброс для VLESS-панели, добавить вторую строку `LocalForward` в тот же блок.

Подключаемся:

```bash
ssh vps
```

В браузере на своём ПК открыть: **http://127.0.0.1:51821/**

## Первый запуск (мастер настройки)

**Где:** в браузере (http://127.0.0.1:51821/)

1. **Username** и **Password** — логин и пароль для входа в панель (задать свои).
2. **Host** — адрес сервера, к которому будут подключаться клиенты: **195.62.49.129** (или домен, если есть).
3. **Port** — порт WireGuard на сервере: **51820** (если не меняли в compose).
4. При первом развёртывании выбрать **No** (новый конфиг). При переносе с v14 — **Yes** и загрузить `wg0.json`.

Сохранить настройки. После этого в панели можно создавать клиентов и скачивать конфиги или QR-коды.

## Firewall — открыть порт 51820/UDP

**Где:** на сервере под super

```bash
sudo ufw allow 51820/udp
sudo ufw status
```

Порт **51821** (веб-интерфейс) в UFW не открывать — доступ только через SSH-туннель.

---

## Добавление клиента и подключение

1. В панели WG-Easy: **Add Client** — имя (например `phone` или `laptop`).
2. Скачать конфиг (**.conf**) или отобразить QR-код.
3. На устройстве: установить приложение WireGuard (Windows, macOS, iOS, Android), импортировать конфиг или отсканировать QR, включить туннель.

Адрес сервера в конфиге будет **195.62.49.129:51820** (или указанный вами Host и Port).

## Обновление WG-Easy

**Где:** на сервере под super

```bash
cd ~/docker/wg-easy
docker-compose pull
docker-compose up -d
```

## Порты (сводка)

| Порт  | Протокол | Назначение    | Доступ из интернета |
| ----- | -------- | ------------- | ------------------- |
| 51820 | UDP      | WireGuard     | Да (ufw allow)      |
| 51821 | TCP      | Веб-интерфейс | Нет (только SSH)    |

## Структура ~/docker (общая с 3X-UI)

На сервере все Docker-сервисы в одном месте:

```
~/docker/
  3x-ui/       # панель VLESS (git clone)
  wg-easy/     # WireGuard + веб-интерфейс (docker-compose.yml)
```

Запуск/остановка: переходить в нужный подкаталог и выполнять `docker-compose up -d` или `docker-compose down`.
