# Настройка VLESS (Xray + XTLS-Reality + 3X-UI)

Продолжение после [vps-setup.md](vps-setup.md) и [docker-services-setup.md](docker-services-setup.md): сервер обновлён, пользователь **super**, SSH на порту **28472**, подключение через `ssh vps`, Docker и базовая структура `~/docker` уже подготовлены.

## Выбор сайта для маскирования (Reality)

- Сайт за рубежом, не РФ, не политика/СМИ.
- Поддержка TLS 1.3 и HTTP/2.
- Главная страница без редиректа на другой домен.
- Желательно: IP сайта близок к IP VPS (одна подсеть/страна).

**Проверка:** зайти на выбранный домен в браузере — открывается без ошибок.

### RealiTLScanner (поиск домена под твой VPS)

Утилита сканирует сайты и выводит домены, подходящие для Reality (TLS 1.3, ALPN). Запускать **на своём компьютере**, не на VPS (сканирование с сервера может привлечь внимание).

**Где:** локально (Windows/macOS/Linux)

1. Скачать бинарник со [страницы релизов](https://github.com/XTLS/RealiTLScanner/releases) под свою ОС (или собрать из исходников: `go build` в клонированном репозитории).

2. Запустить сканирование для IP твоего сервера (195.62.49.129):

   ```bash
   ./RealiTLScanner -addr 195.62.49.129
   ```

   Опционально: сохранить результат в файл, увеличить потоки, другой порт:

   ```bash
   ./RealiTLScanner -addr 195.62.49.129 -out reality-hosts.csv -thread 4
   ```

3. Дождаться окончания. В выводе (или в `out.csv` / `reality-hosts.csv`) смотреть колонку с доменом (cert-domain) — это кандидаты для поля **Dest** в 3X-UI. Выбрать один домен (например из топа списка).

4. В панели 3X-UI при создании VLESS-Reality в поле **Dest (Target)** указать выбранный домен с портом: `выбранный-домен.com:443`.

Если сканер не используешь — можно взять готовые: **www.microsoft.com**, **www.samsung.com**, **yahoo.com** (указать в Dest как `www.microsoft.com:443` и т.д.).

## Установка панели 3X-UI

**Где:** на сервере под super

```bash
cd ~/docker
git clone https://github.com/MHSanaei/3x-ui.git
cd 3x-ui
```

При необходимости зафиксировать версию (например v2.4.5):

```bash
git checkout v2.4.5
```

В `docker-compose.yml` при желании заменить `image: latest` на `image: v2.4.5`. Запуск:

```bash
docker-compose up -d
```

Панель по умолчанию: `http://195.62.49.129:2053/`, логин и пароль **admin** / **admin**.

## Доступ к панели через SSH (проброс порта)

Панель не открывать в интернет. Подключаться только через туннель.

**Где:** локально

В `~/.ssh/config` в блок для **vps** добавить строку (порт 2053 — стандартный порт панели):

```
  LocalForward 2053 127.0.0.1:2053
```

Пример полного блока:

```
Host vps
  HostName 195.62.49.129
  User super
  IdentitiesOnly yes
  IdentityFile ~/.ssh/id_rsa_vps
  Port 28472
  LocalForward 2053 127.0.0.1:2053
```

Подключаемся:

```bash
ssh vps
```

В браузере на своём ПК открыть: **http://127.0.0.1:2053/**

## Безопасность панели 3X-UI

**Где:** в браузере (панель через http://127.0.0.1:2053/)

В панели:

1. **Настройки панели** → **Настройки панели**:

   - **IP-адрес панели** и **Домен прослушивания панели**: `127.0.0.1` (доступ только через SSH).
   - **Порт панели**: сменить с 2053 на свой (например 41873).
   - **Корневой путь URL адреса панели**: например `/panel/`.

2. **Настройки панели** → **Настройки безопасности**: сменить логин и пароль админа.

3. Сохранить и перезапустить панель.

После смены порта панели обновить **LocalForward** в `~/.ssh/config`: оба порта заменить на новый (например `LocalForward 41873 127.0.0.1:41873`). В браузере: **http://127.0.0.1:41873/panel/** (или свой путь).

## Firewall — открыть порт 443 для VLESS

**Где:** на сервере под super

```bash
sudo ufw allow 443/tcp
sudo ufw status
```

Порт 28472 (SSH) уже открыт по vps-setup. Панель 3X-UI снаружи не открываем.

## Маршрутизация в Xray (блокировка торрентов и РФ)

**Где:** в панели 3X-UI (браузер)

1. **Настройки Xray** → **Базовые шаблоны** → **Блокировка конфигураций**: включить **Запрет использования BitTorrent**.

2. **Настройки Xray** → **Правила маршрутизации** → **Добавить правило**:

   - **Domain:** `geosite:category-ru,regexp:.*\.ru$,regexp:.*\.su$`
   - **Outbound Tag:** `blocked`

3. Сохранить настройки и перезапустить Xray.

## Создание VLESS-Reality подключения

**Где:** в панели 3X-UI (браузер)

В панели: **Подключения** → **Добавить подключение**.

1. **Примечания:** например `VLESS-Reality`.
2. **Протокол:** `vless`.
3. **Порт IP:** оставить пустым или указать 195.62.49.129.
4. **Порт:** `443`.
5. **Безопасность:** `Reality`.

В блоке первого **Клиента**:

- Включить.
- **Email:** уникальное имя (например `client_pc`).
- **Flow:** `xtls-rprx-vision`.

В настройках протокола:

- **Протокол передачи:** `TCP (RAW)`.
- **uTLS:** `firefox` или `chrome`.
- **Dest (Target):** домен маскировки (например `www.microsoft.com:443` или из RealiTLScanner).
- Нажать **Get New Cert** (заполнятся приватный и публичный ключ).
- **Sniffing:** включить.

Нажать **Создать**.

## Проверка

1. Включить прокси на клиенте.
2. Открыть https://ipinfo.io/ — должен быть IP 195.62.49.129 и страна VPS.
3. Открыть https://2ip.ru/ или https://yandex.ru/internet/ — должен быть ваш реальный (местный) IP.
4. Проверить YouTube и заблокированные сайты — через прокси; российские сайты — без прокси.
