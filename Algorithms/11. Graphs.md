# Graphs

## Суть и контекст

Обход графа — фундаментальная операция для решения большинства задач на графах: поиск пути, обнаружение циклов, нахождение связных компонент. Два основных алгоритма: DFS (в глубину) и BFS (в ширину). Ключевое отличие от обхода деревьев — необходимость отслеживать посещённые вершины, так как граф может содержать циклы.

## Ключевые идеи

- DFS идёт «глубоко» по одной ветке до конца, потом backtrack
- BFS идёт «широко» — все соседи уровня k перед уровнем k+1
- Граф может содержать циклы → необходим visited set
- Visited set: HashTable, массив или атрибут вершины (цвет: белый/серый/чёрный)
- DFS обнаруживает back edges → цикл в орграфе
- BFS находит кратчайший путь в невзвешенном графе
- Временная сложность обоих: O(V + E)

## Определения и термины

**DFS (Depth-First Search)** — обход в глубину. Начинает с узла, идёт по одной ветке до предела, затем backtrack.

**BFS (Breadth-First Search)** — обход в ширину. Исследует всех непосредственных соседей узла перед переходом к их детям.

**Visited Set** — структура данных (HashSet, массив bool, цвет) для отслеживания посещённых вершин и предотвращения бесконечного цикла.

**Классификация рёбер DFS** — четыре типа:
- **Tree edge** — ребро к WHITE вершине (к непосещённой). Образует DFS-дерево.
- **Back edge** — ребро к GRAY вершине (к предку в текущем DFS-пути). Наличие back edge → цикл в орграфе.
- **Forward edge** — ребро к BLACK вершине, являющейся потомком текущей (d[u] < d[v]).
- **Cross edge** — ребро к BLACK вершине, не являющейся ни предком, ни потомком.

**DFS Tree / Forest** — структура, образующаяся при DFS. Если граф несвязный — получается DFS Forest (несколько деревьев), каждое дерево = связная компонента.

**Скобочная теорема (Parenthesis Theorem)** — время открытия d и закрытия f узла в DFS можно представить как скобки. Если v — потомок u, то интервал [v.d, v.f] полностью вложен в [u.d, u.f]: `(u (v v) u)`.

**Двунаправленный поиск (Bidirectional Search)** — для поиска кратчайшего пути между s и t запускаются два BFS: от s и от t. Когда они сталкиваются, путь найден. Сокращает сложность с O(k^d) до O(k^(d/2)), где k — степень ветвления, d — расстояние.

**Coloring (раскраска вершин)** — три цвета состояния вершины: **white** (не открыта), **gray** (открыта, в текущем пути), **black** (полностью обработана).

**Strongly Connected Components (SCC)** — в орграфе: максимальные подмножества, где каждая пара вершин взаимно достижима.

**Transposed Graph (транспонированный граф)** — граф с теми же вершинами, но все рёбра перевёрнуты.

## Как устроено

### Visited Set для предотвращения циклов

В отличие от деревьев, граф может содержать циклы. Без отслеживания посещённых вершин — риск бесконечного цикла. Реализации visited set:
- **HashTable** — `map[int]bool` или `map[string]bool`
- **Массив** — `visited[n]bool` при вершинах-числах
- **Атрибут вершины (цвет)**: white → gray → black

### DFS — рекурсивная реализация

Рекурсия — наиболее естественная реализация DFS благодаря ветвящейся структуре.

### DFS — итеративная реализация

Рекурсивный DFS → итеративный через явный Stack. Стек заменяет системный стек вызовов.

### BFS для кратчайшего пути в невзвешенном графе

BFS — стандартный алгоритм кратчайшего пути по числу рёбер. Работает потому что остаётся как можно ближе к источнику: кратчайший маршрут обнаруживается раньше, чем более длинные.

Использует очередь (FIFO): узлы обрабатываются в том порядке, в каком были обнаружены.

### Обнаружение циклов через DFS

В орграфе: граф ацикличен (DAG) тогда и только тогда, когда DFS не выдаёт back edges.

**Back edge** — ребро к предку в текущем DFS-дереве, то есть к «серой» вершине (открытой, но не завершённой).

Если при рассмотрении ребра (u, v) вершина v — gray → нашли back edge → цикл.

### Временная сложность O(V + E)

Для обоих алгоритмов:
- Каждая вершина открывается и добавляется в очередь/стек **не более одного раза**
- Каждое ребро в списке смежности просматривается **ровно один раз**

Итого: O(V + E) — оптимальная сложность для задач на графах.

### Связные компоненты через DFS

**Неориентированный граф**: запускаем DFS от каждой непосещённой вершины. Каждый запуск охватывает одну связную компоненту. Итог — DFS Forest с ровно столько деревьями, сколько компонент.

**Орграф — SCC**: алгоритм Косарайю:
1. DFS по исходному графу, записываем время завершения вершин
2. Транспонировать граф (все рёбра перевернуть)
3. DFS по транспонированному графу в порядке убывания времени завершения
4. Каждое DFS-дерево в шаге 3 — одна SCC

### Выбор DFS vs BFS

**Предпочти BFS**: когда цель — найти кратчайший путь или ближайший узел к источнику. BFS находит близкие узлы раньше, не тратя время на глубокие ветки.

**Предпочти DFS**: когда нужно посетить каждый узел. Оба работают, но DFS проще реализовать рекурсивно.

## Детали и нюансы

**Порядок push в итеративном DFS**: push соседей в обратном порядке, если нужен тот же порядок обхода, что и у рекурсивного DFS (последний push = первый pop).

**Временны́е метки DFS**: в алгоритме CLRS каждая вершина получает `d[v]` (время открытия) и `f[v]` (время закрытия). Используются для нахождения SCC и топологической сортировки.

**Граф vs Дерево**: в дереве visited set не нужен (нет циклов). В графе — обязателен.

**Двудольный граф (Bipartite)**: граф, вершины которого делятся на 2 группы без рёбер внутри группы. Проверяется через BFS с раскраской в 2 цвета — если нашли ребро между вершинами одного цвета → не двудольный.

## Сравнения и trade-offs

| | DFS | BFS |
|--|-----|-----|
| Структура | Стек / Рекурсия | Очередь (FIFO) |
| Память | O(V) — глубина стека | O(V) — ширина очереди |
| Кратчайший путь | Нет | Да (невзвешенный) |
| Обнаружение циклов | Да (back edges) | Сложнее |
| Связные компоненты | Да | Да |
| Реализация | Рекурсия (проще) | Итеративная |
| SCC | Да (Косарайю) | Нет |
| Уровневый обход | Нет | Да |

## Примеры

```go
// DFS рекурсивный на графе
func dfsRecursive(graph map[int][]int, node int, visited map[int]bool, result *[]int) {
    if visited[node] {
        return
    }
    visited[node] = true
    *result = append(*result, node)
    for _, neighbor := range graph[node] {
        dfsRecursive(graph, neighbor, visited, result)
    }
}

// DFS итеративный через стек
func dfsIterative(graph map[int][]int, start int) []int {
    visited := make(map[int]bool)
    stack := []int{start}
    var result []int

    for len(stack) > 0 {
        node := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        if visited[node] {
            continue
        }
        visited[node] = true
        result = append(result, node)
        for _, neighbor := range graph[node] {
            if !visited[neighbor] {
                stack = append(stack, neighbor)
            }
        }
    }
    return result
}

// BFS — кратчайший путь в невзвешенном графе
func bfsShortestPath(graph map[int][]int, start, end int) int {
    if start == end {
        return 0
    }
    visited := make(map[int]bool)
    queue := []int{start}
    visited[start] = true
    dist := 0

    for len(queue) > 0 {
        dist++
        size := len(queue)
        for i := 0; i < size; i++ {
            node := queue[0]
            queue = queue[1:]
            for _, neighbor := range graph[node] {
                if neighbor == end {
                    return dist
                }
                if !visited[neighbor] {
                    visited[neighbor] = true
                    queue = append(queue, neighbor)
                }
            }
        }
    }
    return -1 // путь не найден
}

// Подсчёт связных компонент
func countConnectedComponents(n int, adjList map[int][]int) int {
    visited := make(map[int]bool)
    count := 0
    var dfs func(v int)
    dfs = func(v int) {
        visited[v] = true
        for _, neighbor := range adjList[v] {
            if !visited[neighbor] {
                dfs(neighbor)
            }
        }
    }
    for v := 0; v < n; v++ {
        if !visited[v] {
            dfs(v)
            count++
        }
    }
    return count
}

// Обнаружение цикла в орграфе через DFS (раскраска)
const (
    white = 0 // не посещена
    gray  = 1 // в текущем пути
    black = 2 // завершена
)

func hasCycleDirected(graph map[int][]int, n int) bool {
    color := make(map[int]int)
    var dfs func(v int) bool
    dfs = func(v int) bool {
        color[v] = gray
        for _, neighbor := range graph[v] {
            if color[neighbor] == gray { // back edge → цикл
                return true
            }
            if color[neighbor] == white {
                if dfs(neighbor) {
                    return true
                }
            }
        }
        color[v] = black
        return false
    }
    for v := 0; v < n; v++ {
        if color[v] == white {
            if dfs(v) {
                return true
            }
        }
    }
    return false
}

// Проверка двудольности через BFS (2-coloring)
func isBipartite(graph map[int][]int, n int) bool {
    color := make(map[int]int) // 0=uncolored, 1 and -1 = two sides
    for start := 0; start < n; start++ {
        if color[start] != 0 {
            continue
        }
        queue := []int{start}
        color[start] = 1
        for len(queue) > 0 {
            node := queue[0]
            queue = queue[1:]
            for _, neighbor := range graph[node] {
                if color[neighbor] == 0 {
                    color[neighbor] = -color[node]
                    queue = append(queue, neighbor)
                } else if color[neighbor] == color[node] {
                    return false
                }
            }
        }
    }
    return true
}
```

## Дополнительные алгоритмы на графах

### Bellman-Ford — кратчайший путь с отрицательными весами

**Идея**: релаксировать **все** рёбра V-1 раз. После k итераций — корректные пути длиной ≤ k рёбер.

**Обнаружение отрицательного цикла**: если после V-1 итерации ещё возможна релаксация → существует отрицательный цикл.

**Сложность**: O(VE) — медленнее Дейкстры, но работает с отрицательными весами.

```
init: d[s] = 0, d[v] = ∞ для v ≠ s
for i = 1 to V-1:
    for each edge (u, v, w):
        if d[u] + w < d[v]: d[v] = d[u] + w
// Проверка отрицательного цикла:
for each edge (u, v, w):
    if d[u] + w < d[v]: → отрицательный цикл
```

### Prim's Algorithm — Минимальное Остовное Дерево (MST)

**Идея**: жадный алгоритм. Начать с произвольной вершины, расти одно дерево добавлением ближайшего ребра, соединяющего дерево с новой вершиной.

**Инвариант**: на каждом шаге — связное дерево минимального веса.

**Сложность**: O(E log V) с binary heap / priority queue.

```go
func primMST(graph map[int][][2]int, n int) int {
    // graph[u] = [[v, weight], ...]
    inMST := make([]bool, n)
    minCost := make([]int, n)
    for i := range minCost {
        minCost[i] = 1<<31 - 1
    }
    minCost[0] = 0

    // min-heap: [cost, vertex]
    pq := &MinPairHeap{}
    heap.Init(pq)
    heap.Push(pq, [2]int{0, 0})

    total := 0
    for pq.Len() > 0 {
        top := heap.Pop(pq).([2]int)
        cost, u := top[0], top[1]
        if inMST[u] {
            continue
        }
        inMST[u] = true
        total += cost
        for _, edge := range graph[u] {
            v, w := edge[0], edge[1]
            if !inMST[v] && w < minCost[v] {
                minCost[v] = w
                heap.Push(pq, [2]int{w, v})
            }
        }
    }
    return total
}
```

### Kruskal's Algorithm — MST через Union-Find

**Идея**: отсортировать все рёбра по весу. Добавлять рёбра по возрастанию, пропуская те, что создают цикл (Union-Find).

**Сложность**: O(E log E) = O(E log V) (сортировка + Union-Find почти O(1)).

```go
type UnionFind struct {
    parent, rank []int
}

func NewUF(n int) *UnionFind {
    p := make([]int, n)
    for i := range p {
        p[i] = i
    }
    return &UnionFind{parent: p, rank: make([]int, n)}
}

func (uf *UnionFind) Find(x int) int {
    if uf.parent[x] != x {
        uf.parent[x] = uf.Find(uf.parent[x]) // path compression
    }
    return uf.parent[x]
}

func (uf *UnionFind) Union(x, y int) bool {
    px, py := uf.Find(x), uf.Find(y)
    if px == py {
        return false // уже в одной компоненте → цикл
    }
    if uf.rank[px] < uf.rank[py] {
        px, py = py, px
    }
    uf.parent[py] = px
    if uf.rank[px] == uf.rank[py] {
        uf.rank[px]++
    }
    return true
}

func kruskalMST(edges [][3]int, n int) int {
    // edges[i] = [u, v, weight]
    sort.Slice(edges, func(i, j int) bool {
        return edges[i][2] < edges[j][2]
    })
    uf := NewUF(n)
    total := 0
    for _, e := range edges {
        u, v, w := e[0], e[1], e[2]
        if uf.Union(u, v) {
            total += w
        }
    }
    return total
}
```

### Раскраска графа и двудольность

**Раскраска графа (graph coloring)**: назначить каждой вершине цвет так, чтобы никакие соседи не имели одинакового цвета. Хроматическое число χ(G) — минимальное необходимое количество цветов.

**Двудольный граф = 2-раскрашиваемый**: граф двудольный тогда и только тогда, когда он 2-раскрашиваем (не содержит нечётных циклов). BFS с 2-coloring проверяет это за O(V+E).

**Интервальный граф (Interval Graph)**: вершины — интервалы, рёбра — пересечения. Минимальное количество цветов для раскраски интервального графа = максимальное количество одновременно пересекающихся интервалов = задача minMeetingRooms.

## Важно / подводные камни / best practices

- **Всегда** отмечай вершины как visited **до или при добавлении** в очередь/стек, не при извлечении — иначе одна вершина добавится несколько раз
- Для BFS: отмечай visited при enqueue (не dequeue)
- DFS back edge: `color == gray` (в пути), а не `color == black` (уже завершён)
- Если граф несвязный — запускай DFS/BFS от **каждой** непосещённой вершины
- Итеративный DFS может обходить в другом порядке, чем рекурсивный (зависит от порядка push)

#flashcards/algorithms/graphs

Зачем нужен visited set при обходе графа?::Граф может содержать циклы. Без visited set алгоритм попадёт в бесконечный цикл. Отслеживаем посещённые вершины через HashSet, массив или цвет.

Временная сложность DFS и BFS?::O(V + E) для обоих. Каждая вершина посещается не более одного раза, каждое ребро просматривается ровно один раз.

Как DFS обнаруживает цикл в орграфе?::Через раскраску: white (не посещена), gray (в пути), black (завершена). Back edge: ребро к gray вершине → цикл. DAG тогда и только тогда, когда нет back edges.

Чем BFS лучше DFS для поиска кратчайшего пути?::BFS обходит уровень за уровнем → найдёт ближайший узел прежде чем уйти глубоко. DFS может уйти в длинную ветку мимо цели.

Алгоритм нахождения SCC (Косарайю)?::1) DFS исходного графа, фиксируем f[v] (время завершения); 2) Транспонировать граф (перевернуть рёбра); 3) DFS транспонированного в порядке убывания f[v]; 4) Каждое DFS-дерево = SCC.

Как подсчитать связные компоненты через DFS?::Запускать DFS от каждой непосещённой вершины. Каждый запуск обходит одну компоненту. Счётчик запусков = число компонент.

Когда выбрать DFS, когда BFS?::BFS: кратчайший путь, ближайший узел, уровневый обход. DFS: обойти все узлы, обнаружить циклы, SCC, топологическая сортировка.

Как проверить двудольность графа?::BFS + 2-coloring. Каждому узлу назначить цвет, соседям — противоположный. Если нашли ребро между узлами одного цвета → не двудольный.

В чём суть Bellman-Ford и когда он нужен?::Релаксировать все E рёбра V-1 раз. O(VE). Нужен когда граф содержит рёбра отрицательного веса — Дейкстра там некорректен. Также обнаруживает отрицательные циклы.

В чём разница Prim's и Kruskal's MST?::Prim's: растёт одно дерево жадно от стартовой вершины, берёт ближайшее ребро в priority queue, O(E log V). Kruskal's: сортирует все рёбра, добавляет в порядке возрастания пропуская циклы (Union-Find), O(E log E).

Почему двудольный граф = 2-раскрашиваемый?::Двудольный граф не содержит нечётных циклов. 2-раскраска возможна тогда и только тогда, когда нет нечётных циклов. BFS + 2-coloring проверяет это за O(V+E).

Как связаны интервальный граф и задача minMeetingRooms?::Интервалы = вершины, пересечения = рёбра. Минимальное число цветов для раскраски интервального графа = максимальное число одновременно пересекающихся интервалов = минимальное число комнат.

Четыре типа рёбер DFS?
?
1. Tree edge — к WHITE (непосещённой), формирует дерево
2. Back edge — к GRAY (предку в пути), признак цикла
3. Forward edge — к BLACK (потомку), d[u] < d[v]
4. Cross edge — к BLACK (не предок и не потомок)

Что такое скобочная теорема (Parenthesis Theorem) DFS?::Времена d и f можно представить как скобки. Если v — потомок u, то [v.d, v.f] полностью вложен в [u.d, u.f]: (u (v v) u). Интервалы либо вложены, либо не пересекаются.

Что такое Bidirectional Search и в чём преимущество?::Два BFS одновременно: от s и от t. Когда сталкиваются — путь найден. Сокращает сложность с O(k^d) до O(k^(d/2)).
