# Sliding Window

## Суть и контекст

Sliding Window (скользящее окно) — техника нахождения contiguous подмассивов или подстрок, удовлетворяющих заданным условиям. Часто применяется как оптимизация brute-force решений: снижает сложность с O(n²) или O(S! × B) до O(n) или O(B).

Используется когда задача требует отслеживания **непрерывной последовательности** элементов в массиве или строке.

## Ключевые идеи

- Окно определяется двумя указателями: `start` и `end`
- Окно расширяется вправо (`end` двигается) и сжимается слева (`start` двигается)
- Каждый элемент посещается не более двух раз → O(n)
- Существует два вида: фиксированное и переменное окно
- Эффективно для поиска кратчайшего/длиннейшего/суммарного подсегмента по критерию

## Определения и термины

**Sliding Window (скользящее окно)** — метод отслеживания контiguous подмассива/подстроки с помощью двух указателей (`start` и `end`), которые расширяют и сжимают окно по мере обхода.

**Fixed Window (фиксированное окно)** — размер окна постоянен на протяжении всей итерации.

**Variable Window (переменное окно)** — окно расширяется и сжимается динамически.

**Closure (замыкание индекса)** — наименьший индекс, который завершает полный подмассив, содержащий все целевые элементы (применяется в задаче Shortest Supersequence).

## Как устроено

### Фиксированное окно

Размер окна задан константой k. Окно сдвигается на 1 вправо на каждом шаге.

Пример: поиск всех перестановок строки s (длиной 4) в строке b — окно шириной 4 движется по b.

### Переменное окно

Алгоритм (на примере Shortest Supersequence и Maximum Subarray):

1. Инициализировать два указателя `start = 0`, `end = 0`
2. **Расширять** окно — двигать `end` вперёд, добавляя элементы в состояние окна (бегущая сумма, карта частот символов)
3. **Проверять** — удовлетворяет ли текущее окно условию
4. **Сжимать** окно — двигать `start` вперёд, как только условие выполнено (ищем минимальный вариант) или нарушено
5. Отслеживать лучший результат (минимальную длину, максимальную сумму и т.д.) на каждом шаге

## Детали и нюансы

**Maximum Subarray (Kadane's Algorithm)**: использует логику окна — сумма сбрасывается до 0, если становится отрицательной, так как отрицательная последовательность никогда не будет частью оптимального подмассива.

**Longest Substring Without Repeating Characters**: частоты символов отслеживаются в хэш-таблице или bit vector. Окно расширяется до появления дубликата, затем сжимается слева до его удаления.

**Shortest Supersequence (Minimum Window Substring)**: ищется «замыкание» индекса — наименьший индекс, завершающий полный подмассив со всеми целевыми элементами. С использованием min-heap или отслеживания вхождений → O(B log S) или O(B).

## Сравнения и trade-offs

| Вид окна | Размер | Когда применять | Пример |
|----------|--------|-----------------|--------|
| Fixed | Константный k | Поиск перестановок, скользящее среднее | Все анаграммы строки |
| Variable | Меняется динамически | Минимальная/максимальная подстрока/подмассив | Min Window Substring |

## Примеры

```go
// Фиксированное окно — максимальная сумма подмассива длиной k
func maxSumFixed(arr []int, k int) int {
    windowSum := 0
    for i := 0; i < k; i++ {
        windowSum += arr[i]
    }
    maxSum := windowSum
    for i := k; i < len(arr); i++ {
        windowSum += arr[i] - arr[i-k]
        if windowSum > maxSum {
            maxSum = windowSum
        }
    }
    return maxSum
}

// Переменное окно — максимальная сумма подмассива (Kadane's Algorithm)
func maxSubarraySum(arr []int) int {
    maxSum := arr[0]
    currentSum := arr[0]
    for i := 1; i < len(arr); i++ {
        if currentSum < 0 {
            currentSum = arr[i]
        } else {
            currentSum += arr[i]
        }
        if currentSum > maxSum {
            maxSum = currentSum
        }
    }
    return maxSum
}

// Переменное окно — длиннейшая подстрока без повторяющихся символов
func lengthOfLongestSubstring(s string) int {
    freq := make(map[byte]int)
    start := 0
    maxLen := 0
    for end := 0; end < len(s); end++ {
        freq[s[end]]++
        for freq[s[end]] > 1 {
            freq[s[start]]--
            start++
        }
        if end-start+1 > maxLen {
            maxLen = end - start + 1
        }
    }
    return maxLen
}

// Переменное окно — минимальная подстрока содержащая все символы target
func minWindowSubstring(s, t string) string {
    need := make(map[byte]int)
    for i := 0; i < len(t); i++ {
        need[t[i]]++
    }
    window := make(map[byte]int)
    have, total := 0, len(need)
    start := 0
    resStart, resLen := 0, len(s)+1

    for end := 0; end < len(s); end++ {
        c := s[end]
        window[c]++
        if need[c] > 0 && window[c] == need[c] {
            have++
        }
        for have == total {
            if end-start+1 < resLen {
                resStart = start
                resLen = end - start + 1
            }
            window[s[start]]--
            if need[s[start]] > 0 && window[s[start]] < need[s[start]] {
                have--
            }
            start++
        }
    }
    if resLen == len(s)+1 {
        return ""
    }
    return s[resStart : resStart+resLen]
}

// Фиксированное окно — все анаграммы строки p в строке s
func findAnagrams(s, p string) []int {
    if len(p) > len(s) {
        return nil
    }
    need := make([]int, 26)
    window := make([]int, 26)
    for i := 0; i < len(p); i++ {
        need[p[i]-'a']++
        window[s[i]-'a']++
    }
    var result []int
    if need == window {
        result = append(result, 0)
    }
    for i := len(p); i < len(s); i++ {
        window[s[i]-'a']++
        window[s[i-len(p)]-'a']--
        if need == window {
            result = append(result, i-len(p)+1)
        }
    }
    return result
}
```

## Важно / подводные камни / best practices

- Убедись, что задача требует **контiguous** (непрерывной) последовательности — это главный признак применимости
- При фиксированном окне не забывай вычитать элемент, выходящий из окна слева
- При переменном окне отслеживай состояние окна в HashMap или массиве частот
- Пространственная сложность зависит от хранимого состояния: O(1) до O(K), где K — размер алфавита или уникальных элементов

#flashcards/algorithms/sliding-window

Что такое Sliding Window?::Техника нахождения контiguous подмассивов/подстрок с помощью двух указателей start и end. Снижает сложность с O(n²) до O(n).

Чем отличается фиксированное окно от переменного?::Фиксированное — размер k постоянен, сдвигается на 1. Переменное — размер меняется динамически: расширяется вправо, сжимается слева.

Алгоритм переменного окна — 5 шагов::1) Инициализировать start=0, end=0; 2) Расширить окно (end++); 3) Проверить условие; 4) Сжать окно (start++); 5) Обновить лучший результат.

Когда применять Sliding Window?::Когда задача требует контiguous (непрерывной) последовательности в массиве/строке: поиск кратчайшего, длиннейшего, суммарного подсегмента по критерию.

Временная и пространственная сложность Sliding Window?::O(n) по времени — каждый элемент посещается не более 2 раз. O(1)–O(K) по памяти, где K — уникальных элементов/символов в окне.

Логика алгоритма Кэдейна (Kadane)?::Текущая сумма сбрасывается до 0 если становится отрицательной — отрицательная последовательность не может быть частью оптимального подмассива.

Как найти длиннейшую подстроку без повторений?::Расширяем end, храним частоты в map. Как только частота символа > 1 — сжимаем start пока дубликат не исчезнет. Обновляем maxLen.
