# Two Pointers

## Суть и контекст

Техника двух указателей (Two Pointers), также известная как техника «бегуна» (Runner Technique) — использование двух индексов для одновременного прохода по структуре данных (массиву или связному списку). Позволяет избежать вложенных циклов и снизить временную сложность с O(n²) до O(n).

## Ключевые идеи

- Два указателя движутся по одной структуре данных независимо
- Три основных паттерна: быстрый/медленный, с противоположных концов, в одном направлении
- Наиболее эффективна на отсортированных данных или для поиска взаимосвязей между элементами на разных позициях
- Временная сложность O(n), пространственная O(1)
- При необходимости предварительной сортировки — O(n log n)

## Определения и термины

**Runner Technique (техника бегуна)** — второе название техники двух указателей. Один указатель «бежит» впереди другого.

**Fast pointer (быстрый указатель)** — указатель, продвигающийся быстрее (например, на 2 шага за итерацию).

**Slow pointer (медленный указатель)** — указатель, продвигающийся медленнее (на 1 шаг за итерацию).

**Floyd's Algorithm (алгоритм Флойда)** — алгоритм обнаружения цикла с помощью быстрого и медленного указателей.

## Как устроено

### Паттерн 1: Быстрый и медленный указатели (Fast/Slow)

Один указатель движется быстрее другого (например, на два шага, пока медленный делает один).

**Поиск середины списка**: когда быстрый достигает конца, медленный оказывается ровно посередине.

**Обнаружение цикла (алгоритм Флойда)**:
- Если цикл есть — быстрый рано или поздно догонит медленного, они встретятся
- Если цикла нет — быстрый достигнет конца

**Поиск k-го элемента с конца**:
1. Продвинуть первый указатель на k шагов вперёд
2. Оба указателя движутся с одинаковой скоростью
3. Когда первый достигает конца — второй указывает на k-й элемент с конца

### Паттерн 2: Указатели с противоположных концов (Opposite Ends)

Указатели устанавливаются на начало и конец структуры, движутся навстречу друг другу.

**Алгоритм поиска пары с заданной суммой**:
1. Отсортировать массив (если не отсортирован)
2. `first = 0`, `last = len - 1`
3. Пока `first < last`:
   - Если `arr[first] + arr[last] == target` → пара найдена
   - Если сумма меньше target → сдвинуть `first` вперёд
   - Если сумма больше target → сдвинуть `last` назад

**Разворот строки / проверка палиндрома**: символы в начале и конце сравниваются (или меняются местами), пока указатели не встретятся в центре.

### Паттерн 3: Указатели в одном направлении (Same Direction)

Используются для сравнения текущего элемента со всеми последующими или при слиянии.

**Удаление дубликатов без буфера**: `current` итерируется по списку, `runner` проверяет все последующие узлы на совпадение значения и удаляет их. Сложность O(n²) по времени, но O(1) по памяти.

**Слияние отсортированных массивов**: два индекса отслеживают текущие элементы в двух массивах, на каждом шаге выбирается меньший.

## Детали и нюансы

**Минимальная разница между двумя массивами**: после сортировки обоих массивов указатель сдвигается в том массиве, где значение текущего элемента меньше — так минимизируется разность.

**Поиск пересечения списков**: зная разницу в длинах, продвигают указатель в длинном списке вперёд, затем итерируются по обоим синхронно до момента встречи.

**Нахождение старта цикла (Floyd)**:
1. После встречи быстрого и медленного — медленный уходит на голову списка
2. Оба движутся по 1 шагу
3. Место следующей встречи — начало цикла

## Сравнения и trade-offs

| Паттерн | Когда применять | Сложность |
|---------|-----------------|-----------|
| Fast/Slow | Циклы, середина, k-й с конца | O(n) время, O(1) память |
| Opposite Ends | Пары с суммой, палиндромы (отсортированные данные) | O(n) время, O(1) память |
| Same Direction | Дубликаты без буфера, слияние | O(n²) или O(n) зависит от задачи |

## Примеры

```go
// Паттерн: Fast/Slow — обнаружение цикла (алгоритм Флойда)
type ListNode struct {
    Val  int
    Next *ListNode
}

func hasCycle(head *ListNode) bool {
    slow, fast := head, head
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
        if slow == fast {
            return true
        }
    }
    return false
}

// Паттерн: Fast/Slow — нахождение середины списка
func findMiddle(head *ListNode) *ListNode {
    slow, fast := head, head
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
    }
    return slow
}

// Паттерн: Fast/Slow — k-й элемент с конца
func kthFromEnd(head *ListNode, k int) *ListNode {
    first, second := head, head
    for i := 0; i < k; i++ {
        first = first.Next
    }
    for first != nil {
        first = first.Next
        second = second.Next
    }
    return second
}

// Паттерн: Opposite Ends — поиск пары с заданной суммой
func twoSum(arr []int, target int) (int, int) {
    // Массив должен быть отсортирован
    lo, hi := 0, len(arr)-1
    for lo < hi {
        sum := arr[lo] + arr[hi]
        if sum == target {
            return lo, hi
        } else if sum < target {
            lo++
        } else {
            hi--
        }
    }
    return -1, -1
}

// Паттерн: Opposite Ends — проверка палиндрома
func isPalindrome(s string) bool {
    lo, hi := 0, len(s)-1
    for lo < hi {
        if s[lo] != s[hi] {
            return false
        }
        lo++
        hi--
    }
    return true
}

// Паттерн: Same Direction — слияние двух отсортированных массивов
func mergeSorted(a, b []int) []int {
    result := make([]int, 0, len(a)+len(b))
    i, j := 0, 0
    for i < len(a) && j < len(b) {
        if a[i] <= b[j] {
            result = append(result, a[i])
            i++
        } else {
            result = append(result, b[j])
            j++
        }
    }
    result = append(result, a[i:]...)
    result = append(result, b[j:]...)
    return result
}
```

## Важно / подводные камни / best practices

- Убедись, что массив **отсортирован** перед применением Opposite Ends
- При работе с Linked List всегда проверяй `fast != nil && fast.Next != nil` во избежание nil pointer
- Для обнаружения цикла достаточно только факта встречи указателей; для нахождения старта цикла нужен второй проход
- Техника работает за O(1) памяти — это её главное преимущество перед HashSet-подходом
