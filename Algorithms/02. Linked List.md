# Linked List

## Суть и контекст

Linked List (связный список) — фундаментальная структура данных, представляющая последовательность узлов, где порядок определяется **указателями**, а не индексами массива. В отличие от массива, не обеспечивает константного доступа к элементу по индексу: чтобы найти k-й элемент, нужно пройти k узлов.

## Ключевые идеи

- Порядок задаётся указателями, а не смежностью в памяти
- Вставка и удаление у головы — O(1); поиск и вставка в середину — O(n)
- Двусвязный список позволяет удалять узел за O(1) при наличии указателя на него
- Техника fast/slow pointer применима для поиска середины, обнаружения цикла, нахождения k-го элемента с конца
- Слияние двух отсортированных списков — O(n), может быть распараллелено

## Определения и термины

**Singly Linked List (односвязный список)** — каждый узел содержит данные и один указатель `next` на следующий узел.

**Doubly Linked List (двусвязный список)** — каждый узел содержит данные и два указателя: `next` (преемник) и `prev` (предшественник). Поддерживает двунаправленный обход.

**Circular Linked List (кольцевой список)** — вариация, в которой последний узел указывает обратно на голову. В двусвязном: `prev` головы указывает на хвост, а `next` хвоста — на голову.

**Sentinel node (сторожевой узел)** — фиктивный узел в конце списка, упрощающий граничные условия при слиянии.

**Runner Technique** — итерация по списку с двумя указателями одновременно, один из которых опережает другой.

## Как устроено

### Структура узла

Узел — объект, содержащий ключ (данные) и необходимые указатели. В средах без явных указателей узлы реализуются через несколько массивов (один для ключей, один для `next`, один для `prev`), где «указатель» — общий индекс в этих массивах.

### Операции и временная сложность

| Операция | Сложность | Описание |
|----------|-----------|----------|
| Search | O(n) | Линейный обход для поиска ключа |
| Insert (голова) | O(1) | Обновление нескольких указателей |
| Insert (середина/хвост) | O(n) | O(n) на поиск позиции + O(1) на вставку |
| Delete (двусвязный) | O(1) | Если указатель на узел известен — доступ к соседям прямой |
| Delete (односвязный) | O(n) | Нужно найти предшественника для обновления его `next` |

### Fast/Slow Pointer (Runner Technique)

**Поиск середины**: fast двигается по 2 шага, slow — по 1. Когда fast достигает конца, slow находится ровно на середине.

**Перестановка элементов**: применяется для «переплетения» элементов без знания общей длины списка заранее.

### Cycle Detection — алгоритм Флойда

**Обнаружение**: fast двигается со скоростью 2x, slow — 1x. Если они сталкиваются — цикл существует.

**Нахождение начала цикла**:
1. После столкновения — slow уходит на голову, fast остаётся в точке встречи
2. Оба движутся со скоростью 1x
3. Точка следующей встречи — начало цикла

### Разворот связного списка

Разворот односвязного списка:
- O(n) по времени
- O(1) по дополнительной памяти
- Нерекурсивный алгоритм: итерируемся по списку, переназначая `next` каждого узла на его предшественника

### Слияние отсортированных списков

Слияние двух отсортированных списков: O(n), где n — суммарное количество элементов.

Базовый шаг: сравниваем головы обоих списков, удаляем меньший, добавляем в результат.

Для упрощения граничных условий — используется сторожевой узел (sentinel) в конце каждого списка.

В многопоточной среде: слияние можно распараллелить через бинарный поиск для нахождения медианы одного списка в другом и слияния под-диапазонов параллельно.

## Детали и нюансы

**Удаление из двусвязного за O(1)**: при наличии указателя на узел можно напрямую обратиться к соседям (`prev` и `next`) и перешить указатели. В односвязном — даже зная узел, нужен предшественник → O(n) в общем случае.

**Хранение ключей в узлах LRU**: в реализации LRU-кэша узлы двусвязного списка должны хранить и ключ, и значение — чтобы при вытеснении хвостового узла знать, какую запись удалить из HashMap.

**Множественные массивы вместо указателей**: в языках без явных указателей связные списки реализуются через три параллельных массива (key[], next[], prev[]). «Указатель» — это просто общий индекс.

## Сравнения и trade-offs

| | Singly Linked List | Doubly Linked List | Array |
|--|---------------------|---------------------|-------|
| Доступ по индексу | O(n) | O(n) | O(1) |
| Вставка в голову | O(1) | O(1) | O(n) |
| Удаление по указателю | O(n) | O(1) | O(n) |
| Память на узел | ключ + 1 указатель | ключ + 2 указателя | только данные |
| Двунаправленный обход | Нет | Да | Да |

## Примеры

```go
// Структура узла
type ListNode struct {
    Val  int
    Next *ListNode
}

// Двусвязный узел
type DListNode struct {
    Key, Val   int
    Prev, Next *DListNode
}

// Разворот односвязного списка — O(n) время, O(1) память
func reverseList(head *ListNode) *ListNode {
    var prev *ListNode
    curr := head
    for curr != nil {
        next := curr.Next
        curr.Next = prev
        prev = curr
        curr = next
    }
    return prev
}

// Обнаружение цикла (алгоритм Флойда)
func hasCycle(head *ListNode) bool {
    slow, fast := head, head
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
        if slow == fast {
            return true
        }
    }
    return false
}

// Нахождение начала цикла
func detectCycleStart(head *ListNode) *ListNode {
    slow, fast := head, head
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
        if slow == fast {
            slow = head
            for slow != fast {
                slow = slow.Next
                fast = fast.Next
            }
            return slow
        }
    }
    return nil
}

// Поиск середины списка
func findMiddle(head *ListNode) *ListNode {
    slow, fast := head, head
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
    }
    return slow
}

// Слияние двух отсортированных списков
func mergeTwoLists(l1, l2 *ListNode) *ListNode {
    dummy := &ListNode{}
    curr := dummy
    for l1 != nil && l2 != nil {
        if l1.Val <= l2.Val {
            curr.Next = l1
            l1 = l1.Next
        } else {
            curr.Next = l2
            l2 = l2.Next
        }
        curr = curr.Next
    }
    if l1 != nil {
        curr.Next = l1
    } else {
        curr.Next = l2
    }
    return dummy.Next
}

// Удаление дубликатов без буфера — O(n²) время, O(1) память
func deleteDuplicatesNoBuffer(head *ListNode) {
    current := head
    for current != nil {
        runner := current
        for runner.Next != nil {
            if runner.Next.Val == current.Val {
                runner.Next = runner.Next.Next
            } else {
                runner = runner.Next
            }
        }
        current = current.Next
    }
}
```

## Важно / подводные камни / best practices

- Всегда проверяй `head == nil` и `head.Next == nil` как граничные случаи
- При использовании fast pointer: `fast != nil && fast.Next != nil` — порядок важен
- Для удаления узла из односвязного нужен предшественник — помни про O(n)
- Sentinel (dummy) узел упрощает граничные условия при слиянии и вставке в голову
- В LRU — узлы двусвязного списка должны хранить и **ключ**, и значение

## Карточки для повторения

#flashcards/algorithms/linked-list

Что такое Linked List?::Последовательность узлов, порядок которых определён указателями, а не индексами массива. Нет константного доступа по индексу — O(n) для поиска k-го элемента.

Чем singly отличается от doubly linked list?::Singly: каждый узел имеет только next. Doubly: каждый узел имеет next и prev — позволяет удалять узел за O(1) при наличии указателя.

Сложность основных операций Linked List::Search — O(n); Insert голова — O(1); Insert середина — O(n); Delete doubly (по указателю) — O(1); Delete singly — O(n) из-за поиска предшественника.

Почему удаление из doubly быстрее чем из singly?::В doubly у узла есть prev — напрямую перешиваем соседей за O(1). В singly нет prev — нужно найти предшественника за O(n).

Как работает алгоритм Флойда для обнаружения цикла?::Fast двигается по 2 шага, slow по 1. Если встретились — цикл есть. Если fast достиг конца — цикла нет.

Как найти начало цикла через алгоритм Флойда?::1) Обнаружить встречу fast и slow. 2) Slow уходит на head, fast остаётся. 3) Оба по 1 шагу. 4) Следующая встреча = начало цикла.

Как найти середину списка через fast/slow?::Fast двигается по 2 шага, slow по 1. Когда fast достигает конца — slow на середине.

Сложность разворота связного списка?::O(n) по времени, O(1) по памяти. Нерекурсивный алгоритм: переназначаем next каждого узла на предшественника.

Сложность слияния двух отсортированных списков?::O(n), где n — суммарное число элементов. Сравниваем головы, берём меньший. Sentinel node упрощает граничные условия.

Зачем sentinel (dummy) узел?::Упрощает граничные условия — не нужно отдельно обрабатывать вставку в пустой список или начало.
