# Topological Sort

## Суть и контекст

Топологическая сортировка (Topological Sort) — линейное упорядочение вершин ориентированного графа такое, что для каждого ребра (u→v) вершина u стоит перед v в последовательности. Визуально: вершины вдоль горизонтальной линии, все рёбра строго указывают слева направо. Применяется для разрешения зависимостей: порядок сборки, планирование задач, порядок выполнения курсов.

## Ключевые идеи

- Возможна только для DAG (Directed Acyclic Graph) — ориентированного ациклического графа
- Два подхода: алгоритм Кана (in-degree + очередь) и DFS-based (время завершения)
- Обнаружение цикла: встроено в оба алгоритма
- Временная сложность O(V+E) для обоих методов
- Топологических сортировок может быть несколько (если есть вершины с одинаковым in-degree)

## Определения и термины

**Topological Sort** — линейное упорядочение вершин DAG такое, что для каждого ребра u→v, u стоит перед v.

**DAG (Directed Acyclic Graph)** — ориентированный граф без циклов. Единственный тип графа, для которого возможна топологическая сортировка.

**In-degree** — количество входящих рёбер в вершину. Вершина с in-degree = 0 не имеет предшественников.

**Kahn's Algorithm** — алгоритм топологической сортировки на основе in-degree и очереди. Инициирует с вершин без зависимостей (in-degree = 0).

**DFS-based Topological Sort** — при завершении DFS-обхода вершина вставляется в начало связного списка. Список в конце = топологический порядок (обратный порядок времён завершения).

**Circular Dependency (циклическая зависимость)** — цикл в графе зависимостей. Топологическая сортировка невозможна.

**Finishing Time (время завершения)** — момент DFS, когда алгоритм завершил обработку всех потомков вершины. Обозначается f[v].

## Как устроено

### Алгоритм Кана (Kahn's Algorithm)

**Шаг 1 — Инициализация**:
- Вычислить in-degree для каждой вершины
- Добавить в очередь все вершины с in-degree = 0 (нет зависимостей → можно обработать первыми)

**Шаг 2 — Обработка**:
- Пока очередь не пуста: dequeue вершину u → добавить в топологический порядок

**Шаг 3 — Обновление зависимостей**:
- Для каждого потомка v вершины u: уменьшить in-degree[v] на 1
- Если in-degree[v] стало 0 → все предшественники обработаны → добавить v в очередь

**Обнаружение цикла (Kahn's)**:
Если итоговый топологический порядок содержит **не все** вершины — оставшиеся зациклены (их in-degree никогда не достигло 0).

### DFS-based Topological Sort

1. Запустить DFS на графе
2. По мере **завершения** вершины (все её исходящие пути исследованы) → **вставить в начало** связного списка
3. Список в конце = топологический порядок (reverse finishing times)

Почему работает: вершина u вставляется в начало после вершины v только если v завершилась раньше u. Для ребра u→v: u завершится позже v (v — потомок), значит u окажется в списке левее v.

**Обнаружение цикла (DFS-based)**:
При DFS встречается back edge (ребро к «серой» вершине — текущему предку) → цикл найден → топологическая сортировка невозможна.

### Обе сложности — O(V+E)

Каждая вершина и каждое ребро рассматриваются константное число раз.

## Детали и нюансы

**Несколько валидных порядков**: если несколько вершин имеют in-degree = 0 одновременно — любой порядок их обработки даёт валидную топологическую сортировку.

**Kahn's vs DFS**: Kahn's — интуитивнее, явно обнаруживает цикл через подсчёт вершин в результате. DFS — более теоретически элегантный, прямой вывод из свойств DFS.

**Одежда как аналогия**: топологическая сортировка при одевании: носки должны быть надеты до туфель → ребро «носки→туфли». Kahn's: начни с того, что не требует ничего сначала.

## Сравнения и trade-offs

| | Kahn's Algorithm | DFS-based |
|--|-----------------|-----------|
| Структура | In-degree + очередь | DFS + linked list |
| Обнаружение цикла | Конечный подсчёт вершин | Back edge (gray node) |
| Интуитивность | Выше | Ниже |
| Параллелизм | Лучше (уровни можно параллелить) | Сложнее |
| Сложность | O(V+E) | O(V+E) |

## Применения

**Системы сборки (Build Systems)**: определение порядка компиляции пакетов с зависимостями (библиотека A должна быть скомпилирована до приложения B).

**Планирование задач и сборочные линии**: порядок операций на конвейере, где каждая базовая часть должна быть готова до сборки зависящей от неё.

**Расписание курсов**: курс A — предпосылка для курса B → ребро A→B.

**Планирование свадьбы**: зависимые задачи (букет до церемонии и т.д.).

**Повседневная жизнь**: одевание — носки до туфель, трусы до брюк.

## Примеры

```go
// Алгоритм Кана — Topological Sort
func topoSortKahn(n int, edges [][]int) ([]int, bool) {
    inDegree := make([]int, n)
    graph := make([][]int, n)
    for _, e := range edges {
        graph[e[0]] = append(graph[e[0]], e[1])
        inDegree[e[1]]++
    }

    // Инициализация: все вершины с in-degree = 0
    queue := []int{}
    for v := 0; v < n; v++ {
        if inDegree[v] == 0 {
            queue = append(queue, v)
        }
    }

    var order []int
    for len(queue) > 0 {
        u := queue[0]
        queue = queue[1:]
        order = append(order, u)
        for _, v := range graph[u] {
            inDegree[v]--
            if inDegree[v] == 0 {
                queue = append(queue, v)
            }
        }
    }

    // Если порядок содержит не все вершины — цикл
    if len(order) != n {
        return nil, false // цикл обнаружен
    }
    return order, true
}

// DFS-based Topological Sort
func topoSortDFS(n int, graph [][]int) ([]int, bool) {
    const (
        white = 0
        gray  = 1
        black = 2
    )
    color := make([]int, n)
    var result []int
    hasCycle := false

    var dfs func(v int)
    dfs = func(v int) {
        if hasCycle {
            return
        }
        color[v] = gray
        for _, neighbor := range graph[v] {
            if color[neighbor] == gray {
                hasCycle = true // back edge → цикл
                return
            }
            if color[neighbor] == white {
                dfs(neighbor)
            }
        }
        color[v] = black
        result = append([]int{v}, result...) // вставить в начало
    }

    for v := 0; v < n; v++ {
        if color[v] == white {
            dfs(v)
        }
    }

    if hasCycle {
        return nil, false
    }
    return result, true
}

// Задача: порядок курсов (Course Schedule II)
func findOrder(numCourses int, prerequisites [][]int) []int {
    graph := make([][]int, numCourses)
    for _, pre := range prerequisites {
        graph[pre[1]] = append(graph[pre[1]], pre[0])
    }
    order, ok := topoSortKahn(numCourses, prerequisites)
    if !ok {
        return nil
    }
    return order
}

// Задача: порядок сборки пакетов
func buildOrder(projects []string, deps [][2]string) ([]string, error) {
    // Маппинг имя → индекс
    idx := make(map[string]int)
    for i, p := range projects {
        idx[p] = i
    }
    n := len(projects)
    edges := make([][]int, 0)
    for _, d := range deps {
        // d[0] должен быть до d[1]
        edges = append(edges, []int{idx[d[0]], idx[d[1]]})
    }
    order, ok := topoSortKahn(n, edges)
    if !ok {
        return nil, fmt.Errorf("circular dependency detected")
    }
    result := make([]string, n)
    for i, v := range order {
        result[i] = projects[v]
    }
    return result, nil
}
```

## Важно / подводные камни / best practices

- Топологическая сортировка **только для DAG** — при наличии цикла алгоритм обнаружит это
- Kahn's: цикл обнаруживается если `len(result) != n` в конце
- DFS-based: цикл = back edge (ребро к gray вершине)
- Направление рёбер в графе зависимостей: если A требует B → ребро B→A (B перед A в сортировке)
- Несколько валидных ответов — это норма для топологической сортировки
