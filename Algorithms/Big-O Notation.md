# Big-O Notation

## Суть и контекст

Big-O — это метрика и язык для описания эффективности алгоритмов. Вместо измерения скорости в секундах описывает, как количество операций растёт относительно размера входных данных (n). Big-O характеризует **скорость роста** (rate of increase) времени работы алгоритма.

## Ключевые идеи

- Big-O описывает **верхнюю границу** (upper bound) — гарантию наихудшего случая
- Константы и незначительные члены отбрасываются
- Для одного алгоритма существует три случая: наилучший, наихудший, средний
- Временная и пространственная сложности — разные характеристики
- Рекурсивные алгоритмы имеют особую формулу расчёта

## Определения и термины

**Big-O** — нотация, описывающая верхнюю границу роста времени выполнения или памяти алгоритма относительно размера входных данных n.

**Временная сложность (Time Complexity)** — характеристика масштабирования времени обработки.

**Пространственная сложность (Space Complexity)** — объём памяти, требуемый алгоритму. Если создаётся массив размера n — это O(n) памяти.

**BCR (Best Conceivable Runtime)** — абсолютно наилучшее теоретически возможное время выполнения для задачи. Работа ниже BCR считается «бесплатной».

**BUD** — подход к оптимизации: Bottlenecks (узкие места), Unnecessary work (лишняя работа), Duplicated work (дублирование).

## Классы сложности

| Нотация | Название | Пример |
|---------|----------|--------|
| O(1) | Константное | Доступ к элементу массива по индексу |
| O(log n) | Логарифмическое | Бинарный поиск (задача делится пополам на каждом шаге) |
| O(n) | Линейное | Простой линейный поиск |
| O(n log n) | Квазилинейное | Mergesort, Quicksort (в среднем) |
| O(n²) | Квадратичное | Два вложенных цикла, selection sort |
| O(2ⁿ) | Экспоненциальное | Рекурсивный Фибоначчи (время удваивается с каждым элементом) |
| O(n!) | Факториальное | Перебор всех перестановок (Задача коммивояжёра) |

## Как устроено

### Правила расчёта

**1. Отбрасывание констант**
O(2N) → O(N). Big-O описывает скорость роста, а не точное число операций.

**2. Отбрасывание незначительных членов**
O(N² + N) → O(N²). При больших N меньший член становится несущественным.

**3. Алгоритмы из нескольких частей: сложение vs умножение**
- **Сложение**: если алгоритм делает "A единиц работы, затем B единиц работы" → O(A + B)
- **Умножение**: если алгоритм делает "B единиц работы для каждого элемента из A" → O(A × B)

**4. Рекурсивные алгоритмы**
Когда рекурсивная функция делает несколько вызовов, время часто выглядит как:

```
O(ветки^глубина)
```

где `ветки` — количество ветвлений каждого вызова.

### Наилучший, наихудший, средний случай

| Случай | Описание | Применимость |
|--------|----------|--------------|
| **Worst Case** | Максимальное время; даёт гарантию | Big-O обычно описывает этот случай |
| **Expected (Average) Case** | Типичная производительность на случайных данных | Например, Quicksort — O(n log n) в среднем |
| **Best Case** | Минимальное время для «удачного» ввода | Бесполезная метрика на интервью — легко подделать |

### Пространственная сложность

Стековые вызовы рекурсии тоже занимают память. При глубине рекурсии n — это O(n) памяти, так как каждый вызов остаётся в стеке до завершения.

## Детали и нюансы

**Переменные для разных входов**: если в задаче два массива длин A и B — сложность может быть O(A × B), а не O(N²). Использование одной переменной N для разных входов — распространённая ошибка.

**Строки и хэш-функции**: технически поиск строки в хэш-таблице занимает O(K), где K — длина строки, так как нужно прочитать все символы.

**Амортизированная сложность**: вставка с редким расширением массива (resizing) — O(1) амортизированно.

**Константы имеют значение в реальности**: хотя Big-O отбрасывает константы, на практике меньшая константа может сделать O(n log n) алгоритм с Quicksort быстрее другого O(n log n) с Mergesort.

**Неоднозначность переменной n**: важно уточнять — n означает значение числа или количество битов в нём.

## Сравнения и trade-offs

| Подход | Big-O время | Когда применять |
|--------|-------------|-----------------|
| Brute force с вложенными циклами | O(n²) | Отправная точка, потом оптимизируем |
| HashMap для поиска пар | O(n) | Когда нужно убрать внешний цикл |
| Сортировка + два указателя | O(n log n) | Когда данные не отсортированы |
| Бинарный поиск | O(log n) | Только на отсортированных данных |

## Примеры

```go
// O(1) — константное время
func getFirst(arr []int) int {
    return arr[0]
}

// O(n) — линейное время
func linearSearch(arr []int, target int) int {
    for i, v := range arr {
        if v == target {
            return i
        }
    }
    return -1
}

// O(log n) — логарифмическое (бинарный поиск)
func binarySearch(arr []int, target int) int {
    lo, hi := 0, len(arr)-1
    for lo <= hi {
        mid := (lo + hi) / 2
        if arr[mid] == target {
            return mid
        } else if arr[mid] < target {
            lo = mid + 1
        } else {
            hi = mid - 1
        }
    }
    return -1
}

// O(n²) — два вложенных цикла
func hasDuplicateBrute(arr []int) bool {
    for i := 0; i < len(arr); i++ {
        for j := i + 1; j < len(arr); j++ {
            if arr[i] == arr[j] {
                return true
            }
        }
    }
    return false
}

// O(n) — оптимизация через HashMap
func hasDuplicateOptimal(arr []int) bool {
    seen := make(map[int]bool)
    for _, v := range arr {
        if seen[v] {
            return true
        }
        seen[v] = true
    }
    return false
}

// O(2^n) — рекурсивный Фибоначчи
func fib(n int) int {
    if n <= 1 {
        return n
    }
    return fib(n-1) + fib(n-2)
}

// O(n) — Фибоначчи с мемоизацией
func fibMemo(n int, memo map[int]int) int {
    if n <= 1 {
        return n
    }
    if v, ok := memo[n]; ok {
        return v
    }
    memo[n] = fibMemo(n-1, memo) + fibMemo(n-2, memo)
    return memo[n]
}
```

### Сложение vs умножение частей алгоритма

```go
// O(A + B) — сначала A, потом B
func addExample(arrA, arrB []int) {
    for _, v := range arrA { // O(A)
        _ = v
    }
    for _, v := range arrB { // O(B)
        _ = v
    }
}

// O(A * B) — B для каждого элемента A
func multiplyExample(arrA, arrB []int) {
    for _, a := range arrA {   // O(A)
        for _, b := range arrB { // O(B) для каждого
            _, _ = a, b
        }
    }
}
```

## Связи с другими темами

- [[Two Pointers]] — часто снижает O(n²) до O(n)
- [[Sliding Window]] — снижает O(n²) до O(n) для подмассивов
- [[Heap]] — Build-Heap O(n) — классический пример нетривиального анализа; поиск в сбалансированном BST — O(log n)
- [[HashMap]] — ключевой инструмент оптимизации до O(n)
- [[Dynamic Programming]] — мемоизация снижает экспоненциальное до полиномиального

## Важно / подводные камни / best practices

- **На интервью**: сначала озвучь brute force и его Big-O, потом оптимизируй
- **Не угадывай** сложность — выводи её через логику алгоритма
- **Используй BCR**: пойми теоретический минимум и стремись к нему
- **Применяй BUD**: ищи узкие места, лишнюю и дублирующуюся работу
- **Разные переменные**: не называй всё `n` — разные входы требуют разных обозначений

## Карточки для повторения

#flashcards/algorithms/big-o

Что такое Big-O?::Метрика описывающая скорость роста числа операций алгоритма относительно размера входных данных n. Описывает верхнюю границу (worst case).

O(1) — что это и пример?::Константное время. Скорость не зависит от размера входа. Пример: доступ к элементу массива по индексу.

O(log n) — что это и пример?::Логарифмическое время. Задача делится пополам на каждом шаге. Пример: бинарный поиск.

O(n) — что это и пример?::Линейное время. Скорость растёт пропорционально входу. Пример: линейный поиск по массиву.

O(n log n) — что это и пример?::Квазилинейное время. Характерно для эффективных алгоритмов сортировки: Mergesort, Quicksort (в среднем).

O(n²) — что это и пример?::Квадратичное время. Два вложенных цикла. Пример: selection sort, brute force поиск пары.

O(2^n) — что это и пример?::Экспоненциальное время. Время удваивается с каждым новым элементом. Пример: рекурсивный Фибоначчи без мемоизации.

O(n!) — что это и пример?::Факториальное время. Перебор всех перестановок. Пример: решение Задачи коммивояжёра.

Правило отбрасывания констант::O(2N) → O(N). Big-O описывает скорость роста, константы несущественны.

Правило отбрасывания незначительных членов::O(N² + N) → O(N²). При больших N меньший член становится несущественным.

Когда складывать, а когда умножать сложности?::Складывать — если алгоритм делает A работы, ЗАТЕМ B. Умножать — если делает B работы ДЛЯ КАЖДОГО элемента из A.

Формула сложности рекурсивного алгоритма::O(ветки^глубина), где ветки — количество рекурсивных вызовов на каждом уровне.

Чем отличаются Worst/Expected/Best Case?::Worst — максимум (Big-O); Expected — на случайных данных; Best — минимум для «удачного» ввода (бесполезен на интервью).

Что такое BCR?::Best Conceivable Runtime — теоретически наилучшее возможное время для задачи. Работа ниже BCR считается «бесплатной».

Что такое BUD?::Подход к оптимизации: Bottlenecks (узкие места), Unnecessary work (лишняя работа), Duplicated work (дублирование).

Как рекурсия влияет на пространственную сложность?::Каждый рекурсивный вызов занимает место в стеке. При глубине рекурсии n → O(n) памяти.

Типичная ошибка с несколькими входами::Использовать одну переменную N для разных массивов. Если массивы длин A и B — сложность O(A × B), а не O(N²).
