## Суть и контекст

Строка в Go — это срез байтов (`byte slice`), предназначенный только для чтения. Она может содержать байты любого типа и иметь произвольную длину. Go поддерживает строки в формате UTF-8 по умолчанию, что позволяет работать с символами Unicode без дополнительных манипуляций.

UTF-8 была разработана создателями Go — Кеном Томпсоном и Робом Пайком.

---

## Ключевые идеи

- Строка — срез байтов (`[]byte`), иммутабельный (только для чтения).
- `len(s)` возвращает количество **байтов**, а не символов.
- Индексация `s[i]` возвращает байт, а не символ.
- Один символ Unicode может занимать от 1 до 4 байтов.
- Тип `rune` (`int32`) — это кодовая позиция Unicode (code point).
- Цикл `range` по строке итерирует по рунам, а не байтам.
- Чтобы работать со строкой как с массивом символов — привести к `[]rune`.
- Чтобы изменить строку — конвертировать в `[]byte` или `[]rune`, изменить, конвертировать обратно.
- Нулевое значение строки — `""`, не `nil`.
- Срезы строк не копируют данные — обе переменные указывают на один базовый массив.

## Определения и термины

**Строка (string)** — срез байтов, предназначенный только для чтения. Может содержать байты любого типа.

**Иммутабельность** — строки в Go нельзя изменять напрямую по индексу. Чтобы изменить строку, нужно конвертировать в `[]byte` или `[]rune`, изменить и собрать обратно.

**Байт (byte)** — алиас типа `uint8`. Стандартные символы ASCII (цифры, латиница) кодируются одним байтом.

**Руна (rune)** — алиас типа `int32`. Представляет кодовую позицию Unicode (Unicode code point) — то есть символ, а не байт.

**Unicode code point** — уникальный числовой идентификатор символа в стандарте Unicode.

**UTF-8** — кодировка с переменной длиной символа: один символ занимает от 1 до 4 байтов. Разработана Кеном Томпсоном и Робом Пайком.

**`len(s)`** — возвращает количество **байтов** в строке, не количество видимых символов.

**Рунный литерал** — одиночный символ в одинарных кавычках, например `'€'`. Внутреннее представление основано на кодовой позиции Unicode.

**Строковый литерал** — строка, заданная через шестнадцатеричные значения байтов, например `"\x99\x42"`.

## Как устроено

### Хранение в памяти

Строка хранится как последовательность байтов. Обращение по индексу `s[i]` возвращает байт по этой позиции, а не символ.

Пример: строка `"привет"`. Символ «п» кодируется двумя байтами: `D0` и `B7` в шестнадцатеричном виде. Попытка получить `s[0]` вернёт только первый байт — «сырое» числовое значение, которое само по себе не является целым символом «п».

### Кодировка UTF-8

| Количество байтов | Тип символов | Битовый шаблон |
|---|---|---|
| 1 байт | Символы ASCII (коды 0–127) | `0xxxxxxx` |
| 2 байта | Кириллица | `110xxxxx 10xxxxxx` |
| 3 байта | Китайские иероглифы и другие алфавиты | `1110xxxx 10xxxxxx 10xxxxxx` |
| 4 байта | Чаще всего эмодзи | `11110xxx 10xxxxxx 10xxxxxx 10xxxxxx` |

Служебные «high order биты» в начале байта указывают — является ли байт началом нового символа (`110…`, `1110…`, `11110…`) или продолжением предыдущего (`10…`).

### Байты и руны

- **Байты**: соответствуют `uint8`. ASCII-символы занимают 1 байт.
- **Руны**: алиас `int32`. Представляют символ целиком — Unicode code point.
- Чтобы работать с символами, а не байтами, нужно привести строку к `[]rune`.

### Цикл `range` по строке

`range` при итерации по строке возвращает **руны** (не байты) и их начальные позиции в байтах. Это правильный способ перебора строки с символами Unicode.

### Конвертация

| Конвертация | Синтаксис | Когда использовать |
|---|---|---|
| `string` → `[]byte` | `[]byte(s)` | Операции ввода-вывода (пакеты `os`, `io`) |
| `[]byte` → `string` | `string(b)` | После изменения байтов |
| `string` → `[]rune` | `[]rune(s)` | Работа с символами Unicode по индексу |
| `[]rune` → `string` | `string(r)` | После изменения рун |

## Детали и нюансы

### Строки в Go — значения, не указатели

В отличие от C, где строка — это указатель на массив символов с нулевым байтом, в Go строки передаются и хранятся как **значения** (содержащие указатель на байтовый массив и длину). Присваивание строки копирует заголовок (`ptr + len`), но не сами байты.

### `len()` считает байты, а не символы

Если строка содержит кириллицу или другие многобайтовые символы, `len(s)` вернёт количество байтов, которое превысит количество видимых символов.

Пример: строка из 13 символов кириллицы (по 2 байта) и 2 пробелов (по 1 байту) → `len(s)` = 28.

### Срезы строк — не копируют данные

При создании вторичного среза `s2 := s1[1:3]` данные в памяти не копируются — обе переменные указывают на один базовый массив. Это может привести к тому, что большая строка будет удерживаться в памяти, даже если нужен лишь её маленький фрагмент.

### Нулевое значение

Нулевое значение строки — `""` (пустая строка), а **не** `nil`. Строка не может быть `nil`.

### Изменение строки

Строки иммутабельны. Нельзя написать `s[0] = 'A'`. Для изменения нужно:
1. Конвертировать в `[]byte` или `[]rune`.
2. Внести изменения.
3. Конвертировать обратно в `string`.

## Сравнения и trade-offs

| Способ итерации | Итерирует по | Когда использовать |
|---|---|---|
| `for i := 0; i < len(s); i++` | Байтам | ASCII-строки, бинарные данные |
| `for i, r := range s` | Рунам (символам Unicode) | Строки с Unicode, кириллица, эмодзи |

| Тип | Базовый тип | Назначение |
|---|---|---|
| `byte` | `uint8` | Один байт, ASCII-символы |
| `rune` | `int32` | Unicode code point (символ) |
| `string` | `[]byte` (read-only) | Иммутабельная последовательность байтов |
| `[]byte` | Срез байтов | Мутабельные байтовые операции, I/O |
| `[]rune` | Срез рун | Мутабельные операции с символами Unicode |

## Примеры

### 1. Побайтовый вывод строки

```go
for i := 0; i < len(sLiteral); i++ {
    fmt.Printf("%x ", sLiteral[i]) // шестнадцатеричное значение каждого байта
}
```

### 3. Базовая работа с байтами и рунами

```go
s := "привет"
fmt.Println(len(s))       // количество байтов, не символов
fmt.Println(s[0])         // первый байт — не символ «п»

r := []rune(s)
fmt.Println(len(r))       // количество символов
fmt.Println(string(r[0])) // корректный символ «п»
```

### 4. Цикл `range` по рунам

```go
for i, r := range s {
    fmt.Printf("позиция %d: символ %c\n", i, r)
}
```

### 5. Изменение строки через `[]rune`

```go
r := []rune(s)
r[0] = 'П'           // изменить первый символ
s = string(r)        // собрать обратно в строку
```

### 6. Конвертация в `[]byte` для I/O

```go
b := []byte("Mihalis")
// операции с байтами
s := string(b)
```

### 7. Форматирование строк

```go
result := fmt.Sprintf("Имя: %s, возраст: %d", name, age)
```

### 8. Объединение среза строк

```go
parts := []string{"Go", "is", "awesome"}
joined := strings.Join(parts, " ")
```

## Важно / подводные камни / best practices

**Подводные камни:**

- **`len(s)` — байты, не символы.** Для строк с кириллицей, иероглифами, эмодзи `len(s)` вернёт больше, чем количество видимых символов.
- **Индексация `s[i]` — байт, не символ.** Для многобайтовых символов индексация по строке напрямую даёт байт, а не символ.
- **Срезы строк не копируют данные.** `s2 := s1[1:3]` — обе переменные указывают на один базовый массив. Большая строка удерживается в памяти, даже если нужен лишь фрагмент.
- **Иммутабельность.** Нельзя писать `s[0] = 'A'`. Нужна конвертация в `[]byte` или `[]rune`.
- **Строка не может быть `nil`.** Нулевое значение — `""`.

**Best practices:**

- Для работы с символами Unicode (не только ASCII) используй `[]rune` или цикл `range`.
- Для операций ввода-вывода конвертируй строку в `[]byte`.
- Для лексикографического сравнения без учёта регистра используй `strings.EqualFold()`.
- Для частого логирования используй службу журналирования, а не прямой вывод строк.
- Для построения строк с форматированием используй `fmt.Sprintf`.
