# Массивы и Срезы в Go (Arrays & Slices)

## Суть и контекст

**Массив (array)** — структура данных фиксированного размера, хранящая последовательность элементов одного типа в непрерывной области памяти.

**Срез (slice)** — динамическая реализация массива, позволяющая расширять или сужать последовательность элементов. Срез — один из ключевых типов Go, используется повсеместно вместо массивов.

---

## Ключевые идеи

- Массив имеет фиксированную длину; длина является частью его типа — `[3]int` и `[4]int` это **разные типы**.
- Срез — это дескриптор из трёх полей: указатель на базовый массив + длина (`len`) + ёмкость (`cap`).
- Внутри среза **всегда лежит базовый массив**.
- Массив передаётся по значению (копируются все элементы); срез — ссылочный тип (копируется только дескриптор).
- `len()` — количество используемых элементов; `cap()` — сколько можно добавить без перевыделения памяти.
- При нехватке `cap` функция `append` выделяет **новый** массив большего размера и копирует данные.
- Срезы от одного базового массива разделяют данные — изменение одного меняет другой.
- Маленький срез может удерживать в памяти огромный базовый массив — источник утечек.
- Нулевое значение массива — массив с нулями; нулевое значение среза — `nil`.
- Трёхиндексный срез `s[a:b:c]` позволяет ограничить ёмкость и защитить данные.

---

## Определения и термины

**Массив (array)** — последовательность элементов одного типа фиксированной длины. Длина — часть типа. Хранится как непрерывная область памяти. Не является указателем.

**Срез (slice)** — динамическая структура-дескриптор из трёх полей: указатель на базовый массив, длина (`len`), ёмкость (`cap`). Тип среза не включает размер: `[]int`.

**Базовый массив (underlying array)** — непрерывный массив в памяти, на который ссылается срез через внутренний указатель.

**Длина (`len`)** — количество элементов, фактически содержащихся в массиве или срезе. Возвращается функцией `len()`.

**Ёмкость (`cap`)** — максимальное количество элементов, которое может вместить базовый массив, начиная с позиции указателя среза. Возвращается функцией `cap()`. Для массива всегда равна длине.

**Дескриптор среза** — небольшая структура (указатель + len + cap), которая копируется при передаче среза в функцию.

**Релокация (relocation)** — процесс выделения нового базового массива при переполнении `cap` во время `append`. Старые данные копируются в новый массив, указатель в дескрипторе обновляется.

**Трёхиндексный срез (`s[a:b:c]`)** — синтаксис явного задания ёмкости нового среза: `len = b - a`, `cap = c - a`.

**Утечка памяти через срезы** — ситуация, когда маленький срез удерживает ссылку на огромный базовый массив, не давая GC его освободить.

## Как устроено

### Массив в памяти

Массив — непрерывная область памяти. Не является указателем; это структура с адресом начала, длиной и размером элементов. Все элементы хранятся последовательно.

### Срез в памяти

Срез — это дескриптор из трёх полей:

```
┌─────────────┐
│  ptr        │──► [базовый массив в памяти]
│  len        │
│  cap        │
└─────────────┘
```

При передаче среза в функцию копируется этот дескриптор (24 байта), но не данные. Обе копии дескриптора указывают на один и тот же базовый массив.

### Аллокация: стек или куча

| Тип | Условие попадания в стек |
|---|---|
| Массив | Размер < 10 МБ |
| Базовый массив среза | Размер < 64 КБ |

Финальное решение принимает [[Стек и куча]] компилятора.

### Алгоритм роста `append`

1. Если `len < cap` — записывает элемент в следующую свободную ячейку, увеличивает `len` на 1. Аллокаций нет.
2. Если `cap` исчерпан — выделяет новый, более вместительный базовый массив, копирует старые данные, добавляет новый элемент, обновляет указатель в дескрипторе. Старый массив удаляется GC, если на него больше нет ссылок.

**Коэффициент роста ёмкости:**
- Срезы < 256–1024 элементов: ёмкость **удваивается**.
- Срезы > 1024 элементов: ёмкость растёт примерно на **25%** (коэффициент ~1.25).

### Функция `copy`

`copy(dst, src)` — поэлементное копирование. Копирует `min(len(dst), len(src))` элементов.

Важно: если `dst` имеет `len == 0`, `copy` не скопирует ничего. Для полной независимой копии нужно предварительно создать `dst` нужной длины через `make`, или использовать `slices.Clone`.

### Трёхиндексный срез `s[a:b:c]`

- `len = b - a`
- `cap = c - a`

Если задать `b == c`, то `append` внутри функции, получившей такой срез, немедленно вызовет аллокацию нового массива и **не перезапишет** данные за границами под-среза в родительском массиве.

### Двумерные срезы

Двумерный срез — срез, элементами которого являются другие срезы (`[][]int`). При создании через `make([][]int, n)` внутренние срезы равны `nil`. Для добавления элементов используется `append` для каждого внутреннего среза отдельно.

## Детали и нюансы

### Разделяемый базовый массив

`s2 := s1[1:3]` — создаёт новый дескриптор, указывающий на **тот же** базовый массив. Любое изменение элементов `s2` изменит данные в `s1`. Однако если `append` к `s2` превысит его ёмкость, `s2` создаст **новый** базовый массив и «отвяжется» от оригинала.

### Утечка памяти

Маленький срез, ссылающийся на огромный базовый массив, не даёт GC освободить этот массив. Решение — скопировать нужные данные в новый срез с минимальной ёмкостью:

```go
largeData := make([]byte, 1024*1024) // 1 MB
smallCopy := make([]byte, 2)
copy(smallCopy, largeData[0:2])
// теперь largeData может быть собрана GC
```

### Пустой срез vs nil-срез

Пустой срез (`s := []int{}` или `s := make([]int, 0)`) и nil-срез (`var s []int`) — не одно и то же. При сериализации в JSON: `nil`-срез → `null`; пустой срез → `[]`. Проверяй через `len(s) == 0`, а не `s == nil`.
<!--SR:!2026-03-12,1,209-->

### Передача среза в функцию и `append`

При передаче среза по значению и вызове `append` внутри функции:
- Если хватает `cap` — данные в базовом массиве изменяются (и видны снаружи).
- Если `cap` исчерпан — создаётся новый массив; изменения **не видны** снаружи.
- `len` оригинального дескриптора в вызывающей функции **никогда не изменится**.

## Сравнения и trade-offs

| Критерий | Массив | Срез |
|---|---|---|
| Размер | Фиксированный | Динамический |
| Длина как часть типа | Да (`[3]int` ≠ `[4]int`) | Нет (`[]int`) |
| Передача в функцию | По значению (копия всех элементов) | Ссылочный тип (копия дескриптора) |
| Нулевое значение | Массив с нулями | `nil` |
| `cap` | Всегда равна `len` | Может превышать `len` |
| Аллокация (стек) | < 10 МБ | Базовый массив < 64 КБ |

| Способ создания среза | Синтаксис | Особенность |
|---|---|---|
| Литерал | `[]int{1, 2, 3}` | `len == cap == 3` |
| `make` с len | `make([]int, 5)` | `len == cap == 5` |
| `make` с len и cap | `make([]int, 0, 10)` | `len=0`, `cap=10` |
| Срез из массива | `a[1:3]` | Разделяет базовый массив |
| Трёхиндексный | `a[1:3:3]` | Ограничивает `cap` |

## Примеры

### 1. Объявление массивов

```go
var a [5]int              // массив из 5 нулей
a := [3]int{1, 2, 3}     // литерал массива
a := [...]int{1, 2}      // компилятор определит длину
```

### 2. Создание срезов

```go
s := []int{1, 2, 3}          // литерал среза
s := make([]int, 5)           // len=5, cap=5
s := make([]int, 0, 10)       // len=0, cap=10 — пре-аллокация
s := a[1:3]                   // срез из массива
```

### 3. Трёхиндексный срез — защита базового массива

```go
parent := []int{1, 2, 3, 4, 5}
child := parent[1:3:3]  // len=2, cap=2 — append не затронет '4' и '5'
```

### 4. Правильное копирование для предотвращения утечки

```go
largeData := make([]byte, 1024*1024)
smallCopy := make([]byte, 2)
copy(smallCopy, largeData[0:2])
// largeData теперь может быть собрана GC
```

### 5. `append` — всегда присваивать той же переменной

```go
s = append(s, 42)       // правильно
_ = append(s, 42)       // неправильно — результат потерян
```

### 6. Двумерный срез

```go
matrix := make([][]int, 3)   // 3 nil-среза внутри
for i := range matrix {
    matrix[i] = append(matrix[i], 1, 2, 3)
}
```

## Связи с другими темами

- [[Стек и куча]] — массивы хранятся в стеке если < 10 МБ; базовые массивы срезов — если < 64 КБ; escape analysis определяет финальное решение.
- [[Nil values]] — нулевое значение среза `nil`; отличие nil-среза от пустого при JSON-сериализации.
- [[Указатели (ссылки)]] — дескриптор среза содержит указатель на базовый массив; изменения через срез — мутация по адресу.
- [[Сборщик мусора (GC)]] — GC не освобождает базовый массив, пока на него ссылается хотя бы один срез — источник утечек памяти.
- [[Строки]] — строка внутри реализована как read-only `[]byte`; срезы строк разделяют базовый массив аналогично срезам.

## Важно / подводные камни / best practices

**Подводные камни:**

- **Разделяемый базовый массив.** `s2 := s1[1:3]` — оба среза указывают на один массив. Изменение элементов `s2` меняет данные в `s1`.
- **`append` и отвязка.** Если `append` к под-срезу исчерпает его `cap`, он создаст новый массив и изменения перестанут отражаться в оригинале — неожиданное поведение.
- **Утечка памяти.** Маленький срез от огромного массива удерживает весь массив в памяти. Решение — копировать нужные байты в новый срез.
- **`copy` с пустым `dst`.** Если `len(dst) == 0`, `copy` не скопирует ничего.
- **`len` не изменится снаружи.** Передача среза в функцию, где делается `append`, не изменит `len` оригинального дескриптора в вызывающей функции.
- **nil vs пустой срез.** При JSON-сериализации `nil` → `null`, `[]int{}` → `[]`. Не путать при проектировании API.

**Best practices:**

- Пре-аллоцируй срез через `make([]T, 0, cap)`, если заранее известно примерное количество элементов — избегай лишних релокаций.
- Всегда присваивай результат `append` обратно той же переменной: `s = append(s, ...)`.
- Проверяй пустоту через `len(s) == 0`, а не через `s == nil`.
- Для защиты базового массива при передаче в функцию используй трёхиндексный срез `s[a:b:b]` или передавай копию.
- Для полной независимой копии среза: `dst := make([]T, len(src)); copy(dst, src)` или `slices.Clone(src)`.
