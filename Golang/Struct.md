## Суть и контекст

`struct` — ключевой инструмент организации данных в Go. Позволяет группировать связанные данные в единый именованный тип. В отличие от классического ООП, Go не имеет наследования — вместо него используется механизм встраивания (embedding). К структурам привязываются методы через механизм receiver.

---

## Ключевые идеи

- Struct — последовательность именованных полей с типами
- Нет классического наследования — есть embedding для «заимствования» реализации
- Value semantics по умолчанию: структура передаётся по значению (копия)
- Методы привязываются к типу через receiver (value или pointer)
- Пустая `struct{}` имеет нулевой размер (0 байт)
- Теги — метаданные полей, видимые через reflection и игнорируемые компилятором
- При embedding receiver продвинутого метода остаётся встроенный тип, а не внешняя структура

---

## Определения и термины

**Struct** — последовательность именованных элементов, называемых полями. У каждого поля есть имя и тип.

**Поле (field)** — именованный элемент структуры с указанным типом.

**Тег (tag)** — необязательный строковый литерал, следующий за типом поля в объявлении. Становится атрибутом для всех полей в рамках данного объявления.

**Встраивание (embedding)** — объявление поля только по типу, без явного имени. Тип такого поля — T или \*T (где T не может быть указателем или параметром типа). Имя базового типа становится именем поля.

**Продвижение (promotion)** — механизм, при котором поля и методы встроенного типа действуют как поля и методы внешней структуры.

**Value receiver** — метод получает копию структуры; не может изменять оригинал.

**Pointer receiver** — метод получает указатель на структуру; может изменять оригинал.

**Composite literal** — выражение, которое создаёт новый экземпляр структуры каждый раз при вычислении.

**Zero value структуры** — рекурсивное присвоение нулевых значений каждому полю (0 для чисел, false для bool, nil для указателей).

**Exported field** — поле, имя которого начинается с заглавной буквы Unicode (категория Lu); доступно из других пакетов.

**Unexported field** — поле, имя которого начинается со строчной буквы или `_`; недоступно из других пакетов.

**Анонимная структура** — type literal без имени, конструируется на лету.

## Как устроено

### Объявление

```go
type Point struct {
    X, Y float64 // имена полей X и Y типа float64
    name string  // поле name типа string
}
```

- Имена non-blank полей внутри одной структуры должны быть уникальными
- Структура не может содержать поле своего же типа ни напрямую, ни косвенно — только через указатель

### Создание экземпляров

```go
// new(T) — выделяет память, обнуляет, возвращает *T
p1 := new(Point)
p2 := &Point{} // эквивалентно new(Point)

// Composite literals с указанием ключей (в любом порядке)
p3 := Point{X: 10.5, Y: 20.0, name: "A"}
p4 := Point{Y: 20.0} // X и name примут нулевые значения
```

- Без ключей: значения строго в порядке объявления полей, для каждого поля обязательно
- Нельзя передавать значения для неэкспортируемых полей из другого пакета через composite literal

### Теги

```go
type Person struct {
    Name string `json:"name"`
}
```

- Тег — необязательный строковый литерал после типа поля
- Игнорируется компилятором в обычной логике программы
- Видим через интерфейс рефлексии (reflection)
- Участвует в определении идентичности типов структур
- **Игнорируется при конвертации типов**

### Встраивание (Embedding)

```go
type Job struct {
    Command string
    *log.Logger // Анонимное поле (встраивание)
}
```

- Поля и методы встроенного типа «поднимаются» и действуют как поля/методы внешней структуры
- Нельзя использовать имена promoted полей как ключи в composite literal внешней структуры
- **Важно:** когда вызывается promoted метод, receiver — встроенный тип, а не внешняя структура

**Правила набора методов (method set) при встраивании:**
- `S` содержит `T` → method set `S` включает методы с receiver `T`; method set `*S` включает методы с receiver `T` и `*T`
- `S` содержит `*T` → method set `S` и `*S` оба включают методы с receiver `T` и `*T`

**Разрешение конфликтов имён:**
- Поле или метод на верхнем уровне структуры скрывает любой элемент с тем же именем в более глубоко вложенных анонимных полях
- Если дублирующееся имя появляется на одном уровне вложенности — ошибка компиляции (если это имя используется в программе)

### Методы

```go
// Value receiver — получает копию
func (p Point) String() string {
    return fmt.Sprintf("(%v, %v)", p.X, p.Y)
}

// Pointer receiver — может изменять структуру
func (p *ByteSlice) Append(data []byte) {
    // ... реализация
}
```

Метод привязывается к базовому типу receiver, объявленному в том же пакете.

**Синтаксическое удобство:**
- Addressable значение → компилятор автоматически вставляет `&`: `b.Write()` → `(&b).Write()`
- Указатель → компилятор автоматически разыменовывает: `pt.Mv` → `(*pt).Mv`

### Сравнение структур

- Операторы `==` и `!=` применимы только если **все** типы полей являются comparable
- Поля сравниваются по очереди в порядке объявления в исходном коде
- Сравнение останавливается при первом найденном отличии

### Memory Layout

```go
unsafe.Alignof(x)    // минимальное выравнивание переменной в байтах
unsafe.Offsetof(s.f) // смещение поля f относительно начала структуры s
unsafe.Sizeof(x)     // полный размер в байтах
```

- Выравнивание структуры = наибольшее выравнивание среди всех полей, но минимум 1 байт
- Структура без полей имеет размер 0 байт

> В предоставленных источниках не описан конкретный алгоритм компилятора по добавлению padding между полями.

### Пустая структура

```go
type Empty struct{}
var s struct{}
```

- Размер 0 байт
- Спецификация гарантирует: две разные переменные нулевого размера могут иметь один и тот же адрес в памяти

> В предоставленных источниках для реализации множеств (Set) упоминается `map` со значениями типа `bool` (Effective Go). Паттерн `map[K]struct{}` явно не описан.

### Анонимные структуры

```go
var p struct{ x int; y string }
```

Type literal — тип без имени, конструируется на лету. Может использоваться в переменных или composite literals без предварительного объявления через `type`.

### Конвертация между структурами

Разрешена, если структуры имеют одинаковые underlying types. **Теги при этом игнорируются:**

```go
type Person struct {
    Name    string `json:"name"`
    Address *struct {
        Street string `json:"street"`
        City   string `json:"city"`
    } `json:"address"`
}

var data *struct {
    Name    string
    Address *struct {
        Street string
        City   string
    }
}

// Конвертация валидна: теги игнорируются при сравнении типов для конвертации
var person = (*Person)(data)
```

### Идиома «Полезное нулевое значение» (Useful Zero Value)

Проектируй структуры так, чтобы нулевое значение (память, выделенная через `new(T)` и заполненная нулями) было сразу готово к использованию без дополнительной инициализации. Примеры:

- `bytes.Buffer` — готов к записи без вызова конструктора
- `sync.Mutex` — является разблокированным мьютексом в нулевом состоянии

Когда нулевого значения недостаточно — используй функцию-фабрику, возвращающую `&T{field: value}`. Это позволяет за одно выражение аллоцировать память и провести инициализацию.

### Встраивание sync.Mutex

- Нулевое значение `sync.Mutex` — готовый к использованию разблокированный мьютекс
- При встраивании методы `Lock()` и `Unlock()` продвигаются на уровень внешней структуры
- Можно вызывать `c.Lock()` вместо `c.mu.Lock()`
- Receiver для этих методов — встроенный мьютекс, а не внешняя структура

> В источниках прямой пример с `sync.Mutex` не приводится. Демонстрируется аналогичный паттерн со встраиванием `*log.Logger` в структуру `Job`.

### Экспортируемые и неэкспортируемые поля

- **Exported**: имя начинается с заглавной буквы Unicode (категория Lu) — доступно из других пакетов
- **Unexported**: строчная буква или `_` — недоступно за пределами пакета
- Неэкспортируемые имена из **разных пакетов** всегда считаются разными (different)
- При инициализации структуры из другого пакета через composite literal — ошибка компиляции при попытке задать значение для unexported поля

## Детали и нюансы

- Структура не может рекурсивно содержать поле своего же типа — только `*T` (указатель)
- Пропущенные поля в composite literal с ключами автоматически получают zero value
- Без ключей в composite literal — все поля обязательны строго по порядку
- Promoted поля нельзя использовать как имена в composite literal внешней структуры
- При одинаковом уровне вложенности конфликтующих имён — ошибка компиляции (если имя используется)
- Теги учитываются в type identity, но игнорируются при конвертации типов

---

## Сравнения и trade-offs

| Подход | Плюсы | Минусы | Когда использовать |
|--------|-------|--------|--------------------|
| Value receiver | Безопасно: не изменяет оригинал; вызывается от значений и указателей | Копия данных; не подходит для изменения структуры | Небольшие структуры, read-only операции |
| Pointer receiver | Изменяет оригинал; эффективнее для больших структур | Только от указателей (автоматически вставляется `&` для addressable) | Мутирующие методы, большие структуры |
| Embedding vs явное поле | Promoted методы и поля; лаконичный синтаксис вызова | Нельзя использовать promoted имена в composite literal; receiver — встроенный тип | Заимствование реализации без наследования |
| Exported field | Доступно из других пакетов | Нарушение инкапсуляции | Публичный API |
| Unexported field | Инкапсуляция | Недоступно из других пакетов; разные пакеты — разные имена | Внутреннее состояние |

## Примеры

```go
// Объявление структуры
type Point struct {
    X, Y float64
    name string
}

// Создание через new
p1 := new(Point)
p2 := &Point{}

// Composite literals
p3 := Point{X: 10.5, Y: 20.0, name: "A"}
p4 := Point{Y: 20.0} // X = 0.0, name = ""

// Встраивание
type Job struct {
    Command string
    *log.Logger
}

// Pointer receiver
func (p *ByteSlice) Append(data []byte) {
    // ... реализация
}

// Конвертация с тегами
type Person struct {
    Name    string `json:"name"`
    Address *struct {
        Street string `json:"street"`
        City   string `json:"city"`
    } `json:"address"`
}

var data *struct {
    Name    string
    Address *struct {
        Street string
        City   string
    }
}

var person = (*Person)(data)
```

## Важно / подводные камни / best practices

- Структура не может содержать поле своего же типа напрямую — только через указатель
- Нельзя задать значение для unexported поля через composite literal из другого пакета
- Promoted поля нельзя использовать как ключи в composite literal внешней структуры
- При вызове promoted метода receiver — встроенный тип, не внешняя структура
- Конфликт имён на одном уровне вложенности — ошибка компиляции (если имя используется)
- Теги игнорируются при конвертации типов, но учитываются при type identity
- Если хотя бы одно поле не comparable — структура не поддерживает `==`/`!=`
- Проектируй структуры с «полезным нулевым значением»: если zero value готово к использованию — конструктор не нужен
- `&File{}` и `new(File)` эквивалентны: оба выделяют память, обнуляют и возвращают `*File`

Что такое composite literal для struct?:::Выражение, создающее новый экземпляр структуры при каждом вычислении; формат: `Point{X: 1, Y: 2}`

Как работает composite literal без ключей?::Значения должны быть переданы строго в порядке объявления полей; для каждого поля значение обязательно

**Теги и конвертация**

Учитываются ли теги struct при конвертации типов?::Нет, теги игнорируются при конвертации; учитываются только underlying types

**Memory layout**

Что возвращает `unsafe.Alignof(x)` для структуры?::Минимальное выравнивание переменной структуры в байтах; равно наибольшему выравниванию среди всех полей, минимум 1

Что возвращает `unsafe.Offsetof(s.f)`?::Смещение поля f в байтах относительно начала адреса структуры s

Что возвращает `unsafe.Sizeof(x)`?::Полный размер структуры в байтах

Какой размер у пустой структуры `struct{}`?::0 байт

**Trade-offs**

Когда использовать value receiver?
?
Небольшие структуры, read-only операции; когда не нужно изменять оригинал; вызывается как от значений, так и от указателей

Когда использовать pointer receiver?
?
Мутирующие методы; большие структуры (для эффективности); когда нужно изменять саму структуру; вызывается только от указателей (компилятор вставляет & для addressable)

**Edge cases**

Edge case: можно ли задать значение для unexported поля через composite literal из другого пакета?::Нет, это ошибка компиляции

Edge case: можно ли использовать promoted поля как ключи в composite literal внешней структуры?::Нет, это запрещено

Edge case: может ли struct содержать поле своего же типа напрямую?::Нет; только через указатель `*T`

Edge case: когда struct не поддерживает `==`?::Если хотя бы одно поле имеет несравниваемый тип (например, slice, map, func)
