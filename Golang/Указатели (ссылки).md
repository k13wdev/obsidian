# Указатели (ссылки) в Go

## Суть и контекст

Указатели в Go — это переменные, которые хранят в себе адреса памяти других переменных. В контексте языка Go понятия «указатель» и «ссылка» фактически являются синонимами.

Указатель не содержит само значение (число, строку и т.д.), а лишь указывает на ячейку памяти, где эти данные расположены. В отличие от C и C++, в Go запрещены арифметические операции с указателями — их нельзя складывать или умножать.

Указатели применяются для:
- передачи данных в функции с возможностью их изменения (мутации),
- экономии ресурсов при работе с крупными структурами,
- различения нулевого значения типа от ситуации, когда значение вообще не было присвоено.

---

## Ключевые идеи

- Указатель хранит адрес памяти другой переменной, а не само значение.
- В 64-битной системе указатель занимает 8 байт.
- Оператор `&` — получить адрес переменной; оператор `*` — разыменовать (получить значение по адресу).
- В Go **всегда** передача по значению. При передаче указателя копируется 8-байтовый адрес — но через него можно мутировать оригинальные данные.
- Срезы, мапы и каналы — ссылочные типы: они сами содержат внутри указатели на базовые структуры.
- `new(T)` — аллоцирует и обнуляет память, возвращает `*T`. Не то же самое, что `make`.
- Переприсваивание указателя внутри функции (`p = &other`) не изменяет оригинал — меняет только локальную копию.
- Разыменование через `*p = value` изменяет значение по адресу — оригинал меняется.
- Перед разыменованием указателя всегда проверяй `if p != nil`.

## Определения и термины

**Указатель (pointer)** — переменная, хранящая адрес памяти другой переменной. В Go синоним понятию «ссылка».

**Адрес памяти** — уникальный идентификатор ячейки виртуальной памяти, обычно представляемый в шестнадцатеричном формате.

**Оператор `&` (амперсанд)** — унарный оператор, возвращающий адрес переменной. Выражение `p := &v` означает, что `p` хранит адрес переменной `v`.

**Оператор `*` (звёздочка)** — унарный оператор разыменования. Выражение `*p` даёт доступ к значению по адресу, хранящемуся в `p`. Также используется при объявлении типа указателя: `*int`, `*string`.

**Разыменование** — процесс получения или изменения значения, хранящегося по адресу, на который указывает указатель.

**Nil-указатель** — указатель, не ссылающийся ни на какой адрес. `var p *int` создаёт nil-указатель.

**Ссылочные типы** — типы (слайсы, мапы, каналы), которые сами содержат внутри указатели на базовые структуры данных. При передаче по значению копируется «обёртка» с указателем, а не данные.

**`new(T)`** — встроенная функция, аллоцирующая память для типа `T`, обнуляющая её и возвращающая указатель `*T`.

**Мутация** — изменение оригинальных данных через указатель внутри функции.

**Синтаксический сахар** — возможность писать `p.Field` вместо `(*p).Field` при работе с указателем на структуру.

## Как устроено

### Адрес и получение указателя

Каждая переменная расположена по определённому адресу в виртуальной памяти. Оператор `&` возвращает этот адрес:

```go
v := 10
p := &v  // p хранит адрес переменной v
```

### Разыменование

Оператор `*` позволяет читать или изменять значение по адресу:

```go
fmt.Println(*p)  // прочитать значение v через указатель
*p = 20          // изменить v через указатель
```

### Объявление типа указателя

```go
var p *int     // nil-указатель на int
var s *string  // nil-указатель на string
```

### Передача по значению в Go

В Go **всегда** передача по значению. Разница:

- При передаче обычного значения — копируется всё содержимое (вся структура, строка и т.д.). Изменения внутри функции не влияют на оригинал.
- При передаче указателя — копируется 8-байтовый адрес. Поскольку копия адреса указывает на ту же ячейку памяти, функция может мутировать данные по этому адресу.

### Ссылочные типы

Слайсы, мапы и каналы — ссылочные типы, так как они сами содержат внутри указатели на базовые структуры данных. Передача по значению копирует «обёртку» с указателем, но не сами данные.

### Функция `new()`

`new(T)` аллоцирует память для типа `T`, обнуляет её и возвращает `*T`:

```go
p := new(int)  // *int, значение по адресу = 0
```

**Отличие от `make()`:**
- `new()` только обнуляет память, не инициализирует внутренние структуры.
- `make()` используется исключительно для слайсов, мап и каналов — возвращает готовый к работе объект, а не указатель.

### Указатели на структуры

Структуры часто передаются через указатели для экономии ресурсов и возможности изменения полей. Go предоставляет синтаксический сахар: `p.Field` вместо `(*p).Field`.

Распространённая практика — конструкторы, возвращающие указатель:

```go
func NewMyStruct() *MyStruct {
    return &MyStruct{...}
}
```

## Детали и нюансы

### Переприсваивание указателя — ловушка

Если внутри функции присвоить указателю-параметру новый адрес (`p = &anotherObj`), это изменит только локальную копию указателя. Оригинальный указатель в вызывающей функции останется прежним.

Чтобы изменить сам объект, нужно разыменование: `*p = anotherObj`.

### Слайсы и `append` в функции

Слайс — это структура с указателем на массив, длиной (`len`) и ёмкостью (`cap`). Если передать слайс в функцию по значению и сделать в ней `append`:
- Если хватает `cap` — функция может изменить данные в базовом массиве (оригинальные данные изменятся).
- Но длина (`len`) оригинального слайса у вызывающей стороны **не изменится**.

Чтобы `append` отразился на оригинале, нужно передавать указатель на слайс.

### `defer` и именованные возвращаемые значения

Если функция возвращает **именованное** значение, `defer` может изменить это значение после выполнения `return`, но до того как оно будет передано вызывающей функции. Если возвращаемое значение **не имеет имени**, `defer` работает с его копией и не может изменить результат.

### Размер указателя

В 64-битной системе любой указатель занимает 8 байт (64 бита) вне зависимости от типа, на который он указывает.

### Арифметика с указателями

В Go арифметические операции с указателями запрещены (в отличие от C/C++). Для обхода этого ограничения существует пакет `unsafe`, но его следует избегать.

---

## Сравнения и trade-offs

| Критерий | Передача по значению | Передача указателя |
|---|---|---|
| Что копируется | Всё содержимое | 8 байт адреса |
| Оригинал изменяется? | Нет | Да (через разыменование) |
| Скорость для крупных структур | Медленнее (копирование) | Быстрее |
| Безопасность | Выше (нет неожиданных мутаций) | Ниже (риск nil dereference) |
| Читаемость | Проще | Требует внимания |

| Функция | Назначение | Возвращает |
|---|---|---|
| `new(T)` | Аллоцирует и обнуляет память для T | `*T` (указатель) |
| `make(T, ...)` | Инициализирует slice/map/channel | Сам объект (не указатель) |

## Примеры

### 1. Базовое использование `&` и `*`

```go
v := 10
p := &v    // p = адрес v
fmt.Println(*p) // читаем значение: 10
*p = 20    // меняем через указатель
fmt.Println(v)  // v теперь 20
```

### 2. Мутация через указатель — правильно

```go
func change(a *int) {
    *a = 20
}
```

Изменит оригинал: разыменование меняет значение по адресу.

### 3. Переприсваивание указателя — ошибка

```go
func fail(p *Person) {
    p = &Person{Name: "New"}
}
```

Оригинал **не изменится**: `p` — локальная копия указателя.

### 4. Правильное переприсваивание через разыменование

```go
func success(p *Person) {
    *p = Person{Name: "New"}
}
```

Оригинал изменится: мы записываем новое значение по адресу.

### 5. Функция `new()`

```go
p := new(int)  // *int, *p == 0
*p = 42
fmt.Println(*p) // 42
```

### 6. Объявление nil-указателя

```go
var p *int  // p == nil
```

## Важно / подводные камни / best practices

**Подводные камни:**

- **Переприсваивание указателя в функции** (`p = &other`) — изменяет только локальную копию. Для изменения оригинала используй `*p = value`.
- **Nil pointer dereference** — разыменование nil-указателя вызывает немедленную панику. Всегда проверяй `if p != nil` перед разыменованием.
- **Слайс и `append`** — передача слайса по значению с последующим `append` не изменит `len` оригинала. Передавай указатель на слайс, если нужно изменить длину.
- **`defer` с именованными возвращаемыми значениями** — `defer` может изменить именованный результат после `return`. Безымянный результат `defer` изменить не может.
- **`new` vs `make`** — не используй `new` для слайсов, мап и каналов: получишь указатель на нулевую структуру, а не готовый к работе объект.

**Best practices:**

- Простые типы (`int`, `float64`, `string`) передавай по значению — это читаемее и безопаснее.
- Для крупных структур используй указатель — экономит копирование.
- Всегда проверяй указатель на `nil` перед разыменованием.
- Используй конструкторы (`NewMyStruct`) для создания и возврата указателей на структуры.
- Используй заглавные буквы для публичных (экспортируемых) полей структур и строчные — для приватных.
- Избегай пакета `unsafe`, если только не требуется экстремальная производительность с осознанным отказом от безопасности.
