## Суть и контекст

Реляционная алгебра — формальный язык манипулирования данными в реляционной модели. Это коллекция операций, которые принимают отношения в качестве операндов и возвращают отношение в качестве результата. Гарантирует **свойство замкнутости**, позволяя строить вложенные выражения произвольной сложности.

---

## Ключевые идеи

- Реляционная алгебра обеспечивает **замкнутость**: результат любой операции — отношение, которое можно использовать как операнд следующей операции.
- Восемь основных операций: RENAME, WHERE (сокращение), PROJECT (проекция), TIMES (декартово произведение), JOIN (естественное соединение), UNION, INTERSECT, MINUS.
- Дополнительные операции: DIVIDEBY (деление), EXTEND (расширение).
- Теоретико-множественные операции (UNION, INTERSECT, MINUS) требуют **совместимости по типу** операндов.
- TIMES требует **отсутствия общих имён атрибутов** у операндов.
- Проекция (PROJECT) — алгебраический аналог квантора существования (∃). Деление (DIVIDEBY) — аналог квантора всеобщности (∀).

---

## Определения и термины

**Реляционная алгебра** — коллекция операций, которые принимают отношения в качестве операндов и возвращают отношение в качестве результата.

**Свойство замкнутости** — результат любой реляционной операции сам является отношением, что позволяет использовать вложенные реляционные выражения произвольной сложности.

---

## Как устроено

### Операции реляционной алгебры

**RENAME (переименование)** — принимает отношение и возвращает идентичное, где один или несколько атрибутов имеют другие имена. Необходимо для предотвращения конфликтов совпадения имён при соединениях.

**WHERE (сокращение / селекция)** — возвращает отношение с идентичным заголовком, тело которого состоит только из тех кортежей, для которых логическое выражение (условие сокращения) принимает значение TRUE.

**PROJECT (проекция)** — формирует заголовок путём удаления атрибутов, не указанных в списке. Тело формируется путём удаления значений неуказанных атрибутов с последующим устранением дубликатов субкортежей.

**TIMES (декартово произведение)** — принимает два отношения без общих имён атрибутов. Заголовок — объединение заголовков. Тело — все возможные парные объединения кортежей.

**JOIN (естественное соединение)** — заголовок — объединение заголовков a и b. Тело — кортежи, являющиеся объединением тех кортежей a и b, у которых значения общих атрибутов абсолютно равны.

**UNION (объединение)** — два отношения одинакового типа → кортежи, присутствующие в первом, втором или обоих.

**INTERSECT (пересечение)** — два отношения одинакового типа → кортежи, присутствующие одновременно в обоих.

**MINUS (разность)** — два отношения одинакового типа → кортежи первого, отсутствующие во втором.

**DIVIDEBY (деление)** — принимает делимое a, делитель b и отношение-посредник c. Возвращает значения атрибутов X из a, для которых соответствующие значения Y в c включают абсолютно все значения Y из b.

**EXTEND (расширение)** — возвращает отношение, дополненное новым вычисляемым атрибутом (горизонтальное вычисление).

---

## Детали и нюансы

- **TIMES и общие атрибуты**: декартово произведение для отношений с общими именами атрибутов запрещено — требуется обязательное предварительное RENAME.
- **UNION/INTERSECT/MINUS и совместимость**: парсер отклоняет операцию, если операнды не принадлежат строго к одному типу отношения.
- **Деление и квантор всеобщности**: оператор DIVIDEBY спроектирован Э. Коддом для устранения необходимости писать громоздкие циклы или сложные вложенные подзапросы для запросов с "все".

---

## Примеры

```
-- Переименование атрибутов перед декартовым произведением
P RENAME ( PNAME AS PN, WEIGHT AS WT )

-- Деление: номера поставщиков, поставляющих все известные детали
S { S# } DIVIDEBY P { P# } PER SP { S#, P# }

-- Расширение: добавление вычисляемого атрибута
EXTEND P ADD ( WEIGHT * 454 ) AS GMWT
```

## Связи с другими темами

- [[01. Теория множеств]] — теоретико-множественные операции (UNION, INTERSECT, MINUS, TIMES) составляют часть реляционной алгебры.
- [[02. Логика предикатов]] — PROJECT = квантор существования (∃), DIVIDEBY = квантор всеобщности (∀).
- [[04. Реляционная модель, нормализация и SQL]] — комбинация SELECT-FROM-WHERE в SQL соответствует операциям реляционной алгебры: FROM = декартово произведение, WHERE = Restrict, SELECT = Project.
- [[05. Декомпозиция отношений]] — декомпозиция использует операцию PROJECT, рекомпозиция — JOIN.

---

## Важно / подводные камни / best practices

- **Используйте DIVIDEBY** для запросов "все поставщики, поставляющие все детали" вместо громоздких вложенных подзапросов.
- **Обязательно RENAME** перед TIMES при совпадении имён атрибутов.
- **Проверяйте совместимость типов** операндов перед UNION/INTERSECT/MINUS.
